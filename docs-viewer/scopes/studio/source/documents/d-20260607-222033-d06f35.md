# Docs Viewer Generated Read Endpoints

Generated read endpoints return existing Docs Viewer JSON artifacts without mutating source or published files. They are used by the local `/docs/` shell so manage mode can read the local published projection, including non-viewable docs that remain available to management.

All endpoints require `scope=<scope>`. `scope` must be configured in `docs-viewer/config/scopes/docs_scopes.json`.

## Tree Reads

Endpoints:

```text
GET /docs/index-tree?scope=studio
```

Returned data: the raw generated `index-tree.json` payload for the scope, including the docs tree, viewer options, scope metadata, and per-doc tree records.

Payload shape:

```json
{
  "schema": "docs_index_tree_v1",
  "viewer_options": {
    "non_loadable_doc_ids": [],
    "manage_only_tree_root_ids": []
  },
  "docs": [
    {
      "doc_id": "docs-viewer",
      "title": "Docs Viewer",
      "content_url": "/docs-viewer/published/docs/studio/by-id/docs-viewer.json",
      "children": []
    }
  ]
}
```

Used for:

- rendering the Docs Viewer navigation tree
- resolving parent/child relationships in manage mode
- exposing generated `viewer_options` to the browser

## Recent Reads

Endpoints:

```text
GET /docs/recent?scope=studio
```

Returned data: the raw local published `recent.json` payload for the scope. The manage route declares the edited basis used by this endpoint.

Payload shape:

```json
{
  "schema": "docs_recent_v1",
  "basis": "edited",
  "limit": 10,
  "docs": [
    {
      "doc_id": "docs-viewer",
      "title": "Docs Viewer",
      "content_url": "/docs-viewer/published/docs/studio/by-id/docs-viewer.json",
      "timestamp": "2026-07-16 12:34:56",
      "parent_id": "docs-viewer",
      "parent_title": "Docs Viewer"
    }
  ]
}
```

Used for:

- rendering the route-configured Recent panel
- checking whether generated recent-doc metadata is present for a scope

## Document Payload Reads

Endpoints:

```text
GET /docs/doc?scope=studio&doc_id=docs-viewer
GET /docs/doc?scope=studio&doc=docs-viewer
```

Returned data: the raw generated `by-id/<doc_id>.json` payload for the selected doc.

Payload shape:

```json
{
  "doc_id": "docs-viewer",
  "title": "Docs Viewer",
  "added_date": "2026-06-07",
  "last_updated": "2026-06-07",
  "viewer_url": "/docs/?scope=studio&doc=d-20260424-000000-50b63f",
  "parent_id": "docs-viewer-overview",
  "content_html": "<h1>Docs Viewer</h1>\n"
}
```

Validation and resolution:

- `doc_id` must contain only letters, numbers, underscores, and hyphens.
- the doc must exist in the generated scope tree before its payload path is read
- the payload path must match the content URL recorded in the generated tree

Used for:

- loading selected document content in Docs Viewer
- inspecting non-viewable generated docs in local manage mode
- refreshing a selected document after source edits

## Search Reads

Endpoints:

```text
GET /docs/search?scope=studio
```

Returned data: the raw docs-search index JSON at the configured published search location.

Payload shape:

```json
{
  "header": {
    "schema": "search_index_studio_v1",
    "scope": "studio",
    "version": "blake2b-...",
    "generated_at_utc": "2026-06-07T00:00:00Z",
    "count": 1
  },
  "entries": [
    {
      "id": "docs-viewer",
      "kind": "doc",
      "title": "Docs Viewer",
      "href": "/docs/?scope=studio&doc=d-20260424-000000-50b63f",
      "last_updated": "2026-06-07",
      "display_meta": "2026-06-07",
      "search_terms": [],
      "search_text": "docs viewer"
    }
  ]
}
```

Used for:

- Docs Viewer search UI
- manage-mode validation that search artifacts exist after rebuilds

## Semantic Reference Reads

Endpoints:

```text
GET /docs/references?scope=studio
GET /docs/reference-target?scope=studio&target_kind=tag&target_slug=example
```

Returned data:

- `references` returns the raw generated semantic-reference index JSON for the scope
- `reference-target` returns one generated target bucket from `references/by-target/<target_kind>/<target_slug>.json`

Reference index payload shape:

```json
{
  "header": {
    "schema": "docs_semantic_references_index_v1",
    "scope": "studio",
    "count": 0,
    "target_count": 0,
    "generated_at": "2026-06-07T00:00:00Z"
  },
  "targets": []
}
```

Reference target payload shape: generated target bucket JSON for the selected `target_kind` and `target_slug`. The bucket contains the target identity and the generated inbound reference records for that target.

Validation and resolution:

- `target_kind` must contain only letters, numbers, underscores, and hyphens
- `target_slug` must contain only safe slug characters
- missing target files return a not-found error rather than falling through to arbitrary paths

Used for:

- semantic-reference panels and target drilldowns
- local verification of generated reference output

## Error Behavior

Generated read endpoints reject unsupported scopes, unsafe ids, missing published files, invalid JSON, and unexpected payload paths. They do not write source, published data, activity logs, or watcher-suppression files.
