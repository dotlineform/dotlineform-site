# Share Sub-Scope Documents

## Purpose

Allow an explicitly selected Docs Viewer sub-scope to participate in document sharing as a document corpus.

The first possible use is the Analysis `tags` sub-scope. Those records are descriptive documents about tags, not the canonical tag registry, aliases, or Work and Series assignments. Sharing them must not recreate Tag Data Sharing.

## Possible Capability

- Select the Analysis `tags` sub-scope as a document corpus.
- Export its tag documents through a supported document-sharing profile.
- Optionally attach a read-only Studio tag projection as package context, joined through stable tag ids.
- Accept returned changes only as one complete, valid package of descriptive documents.
- Leave canonical tag vocabulary, alias, and assignment changes as deliberate manual Studio edits.
- Reuse the same document capability for another real sub-scope need rather than inventing tag-specific exchange machinery.

## Boundaries

- This is an extension of document sharing, not a tag adapter, generic domain registry, or new Data Sharing subsystem.
- A package selects one explicit configured sub-scope; it is not an arbitrary filesystem or nested-scope browser.
- Returned rows remain inspection-only. Missing, extra, duplicate, or invalid documents reject the complete package.
- A returned path requires an explicit Docs Viewer source/write boundary for sub-scope documents before implementation can be proposed.
- Optional Studio tag context is read-only package context and never becomes an automatic canonical apply target.
- No automatic tag creation, alias change, assignment update, reconciliation, or promotion workflow.
- No change to the public Analysis report, sub-scope manifest, or existing document publication model merely to enable sharing.

## Current Position

Analysis already publishes its `tags` sub-scope through a parent report, a minimal manifest, and normal sub-scope by-id document payloads. Current document sharing does not claim sub-scope selection or return support.

This feature is paused as a future placeholder. It has no delivery document and places no work inside the [Document Sharing Ownership Cutover](/docs/?scope=studio&doc=d-20260720-172819-93d6b2).
