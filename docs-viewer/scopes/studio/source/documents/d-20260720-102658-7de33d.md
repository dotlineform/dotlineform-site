# Inline Mermaid Rendering Delivery

## Outcome

A document in a repo-backed `local` scope renders fenced Mermaid after its generated HTML is mounted. The canonical source remains inline Markdown. Public and `local_external` scopes continue to receive only the existing static SVG media workflow.

This document identifies a likely delivery boundary. It is not yet an implementation-ready tracker; the browser dependency and mount contract should be confirmed first.

## Likely Blast Radius

| area | likely owner or evidence | expected change |
| --- | --- | --- |
| Markdown output | `studio/shared/python/markdown_renderer.py` | preserve the existing `language-mermaid` code class; add a regression contract rather than a server-side transform |
| Scope eligibility | generated browser scope config and `docs-viewer-config-controller.js` | expose/use the active record's existing `scopeType`; no new per-scope feature flag initially |
| Document mount | `site/docs-viewer/runtime/js/shared/docs-viewer-document-controller.js` | invoke one post-mount inline-diagram adapter with explicit scope context |
| Browser renderer | new focused shared Docs Viewer module plus a repository-versioned browser asset | lazy-load once, render eligible fences, contain failures, and avoid duplicate rendering |
| Styling | shared Docs Viewer content CSS | responsive diagram host, loading state, and visible error/fallback treatment |
| Static Mermaid build | existing CLI producer and `.mmd` to SVG pipeline | no behavior change |
| Static HTML export | `docs-viewer/services/docs_static_html_export.py` | no conversion or bundle addition; prove the inline code block is retained |
| Source-faithful exports | Docs Viewer document export transforms and package contracts | no conversion; prove canonical Markdown remains byte-faithful |
| Embedded content | sub-scope reports, review/detail surfaces, and other `content_html` mounts | excluded initially unless the host supplies the content scope's explicit `scope_type` |
| Verification | focused Python/browser-module tests and one local proof document | scope gate, lazy loading, navigation remount, multiple diagrams, failure containment, and export preservation |

## Dependency Gate

Before implementation, choose one repository-owned browser distribution for the already pinned Mermaid version. The delivery must not:

- import from `node_modules/` through a browser URL;
- load from a CDN;
- add a second independently floating Mermaid version;
- expose the CLI, Puppeteer, or Node boundary to the reader.

If producing a browser asset requires a small build/copy step, that step must have one clear source package, checked output location, and update command. Do not introduce a general frontend bundler solely for this feature.

## Delivery Checkpoints

### IMD-0 — Contract And Dependency Fit

- Confirm the `language-mermaid` HTML emitted by the shared Markdown renderer.
- Confirm `scope_type` is available when the selected document mounts.
- Choose the browser asset path and exact version ownership.
- Prove that public and external-local route bundles do not load the renderer.
- Stop if the feature requires a broad runtime or build-system refactor.

### IMD-1 — Focused Renderer

- Add one lazy inline-Mermaid adapter with strict initialization.
- Render all eligible fences after a normal document mount.
- Preserve accessible source/fallback behavior and contain one broken diagram without failing the document.
- Handle navigation and repeated mounts without duplicate diagrams or leaked state.

### IMD-2 — Scope And Surface Boundary

- Enable only when the active content scope is exactly `local`.
- Prove `public` and `local_external` fences remain ordinary code blocks and load no Mermaid asset.
- Keep embedded report/review/detail mounts excluded unless their scope context is explicit and tested.
- Add one useful Studio inline diagram as the proof document.

### IMD-3 — Export Regression And Closeout

- Prove source-faithful package exports retain exact inline Markdown.
- Prove static HTML export retains the generated code block without SVG generation or a Mermaid bundle.
- Update the durable Document Diagrams and Export owners with shipped behavior.
- Close only after responsive, theme, keyboard/reading, navigation, and error-state browser checks pass.

## Explicitly Deferred

- pre-rendering inline fences during static HTML export;
- choosing rendered-versus-inline behavior per export profile;
- public publication conversion;
- automatic extraction to canonical `.mmd` and SVG;
- inline editing widgets or live source-editor preview;
- rendering inside reports, review packages, or embedded detail documents without an explicit second delivery need.

The [concept](/docs/?scope=studio&doc=d-20260720-102658-b55348) owns the product and export decisions. The shipped [Mermaid Document Diagrams Delivery](/docs/?scope=studio&doc=d-20260718-214206-27df44) remains the implementation authority for separate `.mmd` source and published SVG.
