# LLM-Assisted Tag Development

## Outcome

Studio can supply enough tag and catalogue context for useful ad-hoc questions such as:

- expand the working definition of this tag;
- suggest tags that may be missing from these Works;
- compare related tags or propose aliases.

The response supports human judgement. It is not an automated canonical update.

## Intended Workflow

Studio supplies stable tag and Work ids with the minimum useful source context. A person reviews the response and manually edits either:

- Studio-owned canonical tag or assignment data; or
- a richer tag document in the Docs Viewer tag sub-scope.

The eventual transport may be a direct LLM API call or a simple copy/export action. The first useful query should determine that choice.

## Boundaries

- No generic Data Sharing adapter or returned-package workflow.
- No automatic apply, promotion, or reconciliation path.
- No assumption that an LLM response is canonical, complete, or schema-valid.
- Preserve tag and Work ids as the join between compact catalogue data and richer explanatory documents.
- Keep prompts, model-specific options, and stored provenance proportionate to an actual retained workflow.

## Current Position

This is a possible direction, not an implementation commitment. Manual use of an external LLM remains a valid complete workflow.
