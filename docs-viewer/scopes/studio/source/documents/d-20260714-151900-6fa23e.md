# Index Multiple Selection

## Purpose

Add explicit checkbox selection to the manage-mode index, then let deliberately designed batch actions consume that selection without changing the displayed document.

This is the feature parent. It does not act as an implementation tracker.

## Current State

The checkbox-selection foundation is complete. The manage tree exposes **Select**, a sibling checkbox gutter, Shift-range selection, a selected count, **Select all**, **Clear**, and **Done**. **Select all** replaces the checked set from the complete manage-index model, including collapsed and non-viewable documents; it does not inspect rendered rows. `docs-viewer-index-selection.js` independently owns checked ids, the range anchor, mode, bulk selection, and reload/lifecycle reconciliation. Existing actions still resolve their prior targets except **Prepare package**, whose Action and compact package workflow use checked ids as their only document target. [Runtime Module Ownership](/docs/?scope=studio&doc=d-20260605-125108-1d1ef3) records the shipped foundation.

**Prepare package** is the first shipped consumer. Its [completed delivery](/docs/?scope=studio&doc=d-20260722-133457-e90e21) covers Action registration, selection-only target resolution, the compact workflow, and dedicated-route retirement; [Missing-Summary Package Preparation Delivery](/docs/?scope=studio&doc=d-20260722-165733-88d9ff) adds complete-index **Select all**, modal package filters, and one effective target count. [Prepare Package](/docs/?scope=studio&doc=d-20260722-151224-7b61c4) is the user workflow. Export, Move, and Delete still require separate delivery decisions, so this feature parent remains in progress. Ordinary Import is not a selection consumer, and Returned packages remains a whole-package scope workflow with no index-selection input.
