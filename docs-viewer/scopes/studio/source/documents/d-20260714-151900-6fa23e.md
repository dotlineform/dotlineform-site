# Index Multiple Selection

## Purpose

Add explicit checkbox selection to the manage-mode index, then let deliberately designed batch actions consume that selection without changing the displayed document.

This is the feature parent. It does not act as an implementation tracker.

## Feature Documents

- [Concept](/docs/?scope=studio&doc=d-20260714-234030-876066) — desired interaction, product boundaries, and deferred choices.
- [Architecture](/docs/?scope=studio&doc=d-20260714-234030-f30d01) — proposed state, ownership, action-target, drag/drop, and service boundaries.
- [Delivery](/docs/?scope=studio&doc=d-20260721-074902-838446) — shipped checkbox-selection foundation, reload behavior, and acceptance evidence.
- [Docs Viewer Roadmap](/docs/?scope=studio&doc=d-20260714-234030-1e0af6) — scheduling, dependencies, and suggested feature order.

## Current State

The checkbox-selection foundation is complete. The manage tree exposes **Select**, a sibling checkbox gutter, Shift-range selection, a selected count, **Clear**, and **Done**. `docs-viewer-index-selection.js` independently owns checked ids, the range anchor, mode, and reload/lifecycle reconciliation; current actions still resolve the displayed or explicitly invoked document. [Runtime Module Ownership](/docs/?scope=studio&doc=d-20260605-125108-1d1ef3) records the shipped boundary.

This feature parent remains in progress only because group movement has not been delivered. Group drag/drop, plural move service behavior, Document Sharing UI Redesign, and any other multi-document consumer require separate approved deliveries.
