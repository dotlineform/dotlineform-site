# Index Multiple Selection

## Purpose

Add explicit checkbox selection to the manage-mode index, then let deliberately designed batch actions consume that selection without changing the displayed document.

This is the feature parent. It does not act as an implementation tracker.

## Feature Documents

- [Concept](/docs/?scope=studio&doc=d-20260714-234030-876066) — desired interaction, product boundaries, and deferred choices.
- [Architecture](/docs/?scope=studio&doc=d-20260714-234030-f30d01) — proposed state, ownership, action-target, drag/drop, and service boundaries.
- [Delivery](/docs/?scope=studio&doc=d-20260721-074902-838446) — proposed checkbox-selection foundation, reload behavior, and bounded implementation checkpoints.
- [Docs Viewer Roadmap](/docs/?scope=studio&doc=d-20260714-234030-1e0af6) — scheduling, dependencies, and suggested feature order.

## Current State

The action-target owner is complete. `docs-viewer-action-definitions.js` gives current controls one code-owned target resolver without adding checkbox-selection state. [Runtime Module Ownership](/docs/?scope=studio&doc=d-20260605-125108-1d1ef3) is the durable owner for that shipped boundary.

Its current one-document selection context is not the proposed checkbox interaction.

Checkbox selection, group drag/drop, plural move service behavior, and additional multi-document consumers are not implemented.

The proposed delivery is bounded to the visible checkbox-selection foundation. Group movement and Document Sharing UI Redesign remain separately gated follow-ons, so this delivery document does not imply that either is implemented or authorized.
