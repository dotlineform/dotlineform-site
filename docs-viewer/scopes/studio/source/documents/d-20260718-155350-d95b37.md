# Analytics App Retirement

## Outcome

The repository no longer carries a separate Analytics application after its implemented responsibilities have moved to the applications that own the underlying data and workflows.

Tags belong to Studio. Document packages belong to the repo-integrated Docs Viewer. Surfacing data, commentary, static figures, or bounded runtime visualisations can use Docs Viewer when a concrete feature requires them; those outputs do not justify a generic analysis application.

Tag Data Sharing is retired rather than moved. Future tag development remains manual Studio editing or separately designed advisory assistance with human writeback.

## Retirement Boundary

Retire the Analytics:

- browser shell, route templates, controllers, and runtime configuration;
- local HTTP server and API adapters;
- Data Sharing gateway routes and cross-domain registry projection;
- tag-package adapters, prepare/returned workflows, and their tests;
- local runner, route documentation, and app-specific tests that no longer protect an owned capability.

Move reusable document behavior to Docs Viewer before removing the old host. Retire the generic repository Data Sharing subsystem after its document-only helpers move; there is no remaining second domain owner. Do not retain an empty Analytics shell, compatibility aliases, proxies, forwarding routes, or a generic exchange framework.

## Preserved Responsibilities

- Studio keeps canonical tag identities, assignments, validation, and editing workflows. It does not inherit tag package or returned-package behavior.
- Integrated Docs Viewer keeps supported document-package workflows and any direct Studio catalogue/tag projection used by semantic references.
- The public `/analysis/` scope remains a Docs Viewer knowledge surface. Its name does not imply an Analytics server or analytical engine.
- Mermaid remains an unrelated authored-diagram feature.

## Current Position

`analytics-app/` still hosts active tag tools and Data Sharing routes, and `data-sharing/` still contains both document and tag adapters. This feature defines the retirement outcome only; it is not evidence that those responsibilities have moved or that either source tree is currently removable.
