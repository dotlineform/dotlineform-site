# Docs Viewer Roadmap

## Current Focus

Document Diagrams, Word Document Import, Scope-First Storage Layout, Share Document Packages, Document Sharing UI Redesign, and Local Tree Move Projection are shipped. The Index Multiple Selection checkbox foundation and its Prepare package consumer are shipped; other consumers remain separately gated. The remaining pre-transition work is Scope Media Management inventory, while the separate Analytics App Retirement dependency waits only on Tag Management In Studio. Docs Viewer Core Refactor follows that retirement boundary; Standalone Docs Viewer remains its first independent consumer.

Bug fixes, regressions, safety work, and small maintenance continue normally. [Docs Viewer Manage Theme Restoration](/docs/?scope=studio&doc=d-20260723-101224-e62173) and [Docs Viewer Modal Consistency](/docs/?scope=studio&doc=d-20260723-091111-aaa2d6) are shipped; the latter retains the Admin-hosted UI Workbench for rapid production-backed modal review. Neither maintenance feature blocks Core Refactor. [Public Docs Viewer Mobile Experience](/docs/?scope=studio&doc=d-20260723-191039-f8cb88) is a separate proposed review of the genuinely public reader; local narrow-layout checks do not imply mobile support for Python-backed applications. Other paused features remain visible after Standalone Docs Viewer for later reconsideration.
