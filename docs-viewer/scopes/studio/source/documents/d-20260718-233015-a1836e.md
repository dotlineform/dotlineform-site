# Word Document Import

## Purpose

Let Docs Import accept staged Word `.docx` documents without making Word HTML, Pandoc, or document-fidelity logic part of Docs Viewer.

The conversion path is deliberately narrow:

```text
.docx -> Mammoth semantic HTML -> existing Docs Import HTML-to-Markdown path
```

## Feature Contract

- Register `.docx` as an ordinary Docs Import document source and pin Mammoth in the repository Python dependencies.
- Keep Mammoth focused on Word-to-HTML conversion, with external file access disabled.
- Treat the existing HTML converter as authoritative: unsupported Word or HTML structures may be flattened or discarded unless a serious content-loss case justifies a focused extension.
- Surface useful Mammoth conversion warnings through the existing import result.
- Reuse current supported embedded-image planning and publication without adding a new image pipeline.
- Reuse the existing preview, Markdown validation, fresh document identity, source-create, and rebuild boundaries.

## Boundary

- no Pandoc or separate Word-export-HTML cleanup path;
- no visual-fidelity or round-trip guarantee;
- no special support for comments, tracked changes, equations, custom style inference, Word metadata, or other advanced document features;
- no new import modal or format-specific browser workflow;
- no broader image-format or image-optimisation work unless a representative import proves it necessary.

## Success

Representative Word documents containing ordinary prose, semantic headings, lists, links, simple tables, and optionally an already supported image format import as valid canonical Markdown. Unsupported structures degrade safely, conversion warnings remain visible, and the ordinary Docs Import flow retains write authority.

The shipped proof used a Word-authored document with Title, Subtitle, Heading 1 through 4, prose, emphasis, ordered and unordered lists, a link, a table, and two embedded images. The managed isolated import published canonical Markdown, byte-verified both JPEG package parts, rebuilt document and search output, and returned no conversion bodies to the browser.
