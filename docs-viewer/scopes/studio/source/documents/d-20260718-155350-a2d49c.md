# Tag Management In Studio

## Outcome

Studio owns the canonical tag vocabulary, aliases, groups, work and series assignments, and the local tools used to maintain them.

Tags are catalogue data about Works and Series. Keeping their source, validation, editors, and catalogue identities together removes an application boundary without changing the tag model.

## Boundaries

- Preserve stable tag, Work, and Series ids and their existing relationships.
- Move the current tag editors, validated write paths, coverage views, and canonical tag sources into Studio rather than redesigning them during the ownership move.
- Keep public catalogue output as a projection of Studio-owned source rather than a second tag authority.
- Keep descriptive tag documents in a Docs Viewer sub-scope. They may retain a tag-id relationship without becoming the canonical assignment store.
- Do not carry Data Sharing routes, document-package behavior, or a generic analysis framework into Studio with the tag tools.

## Current Position

The current tag data, services, and routes live under `analytics-app/`. This feature records the target product owner only; no source, route, or runtime migration has been completed.
