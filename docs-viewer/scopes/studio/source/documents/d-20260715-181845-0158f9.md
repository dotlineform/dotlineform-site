## Status

- The static-media foundation is shipped. DIA-1 established the locked Mermaid CLI toolchain and registered provider-neutral producer; DIA-2 registered Studio source and connected full and targeted builds; DIA-3 shipped staged author intake, rendered source links, and changed-identity watcher rendering; DIA-4 shipped the proof documents and durable workflow guidance.
- Local-only inline Mermaid rendering is proposed as a separate delivery. It keeps the fence in canonical Markdown, renders only for `scope_type: local`, and initially leaves all exports inline rather than adding SVG conversion.
- Diagram detail view is proposed as a sibling mini-feature. Persistent SVG can open its stable served URL independently; inline Mermaid joins the same diagram-host and control contract only after its browser renderer ships.
- Staged `.mmd` intake uses **Add image…**, backed by the repository-pinned Mermaid CLI and one registered producer for interactive, targeted-document, watcher, and full Docs builds.
- The source/publication refactor established the convention: canonical `.mmd` belongs in configured source `media/mermaid/`, and derived SVG is written directly to that scope's configured published `svg` location.
- Studio now configures `media/mermaid/` as source for the registered `mermaid` producer. Full builds render every configured source; targeted document builds render only Mermaid-backed SVG identities referenced by the selected documents before their payloads.
- **Add image…** now accepts staged `.mmd`, renders before configured writes, retains the staged source, copies verified canonical source into configured `media/mermaid/`, publishes the verified SVG, and inserts only its SVG token into the dirty Markdown buffer.
- Rendered manage-mode Document Info shows a **Diagrams** section whose diagram-title links open verified Mermaid-backed sources in VS Code. Source-mode info remains the semantic-token picker, and the watcher renders only a changed Mermaid identity.
- [Configuration And Extension Points](/docs/?scope=studio&doc=d-20260512-142027-54b354) and [Docs Import](/docs/?scope=studio&doc=d-20260424-000000-fc8b5d) contain the first accessible, scope-owned proof diagrams. Rendered Document Info links their verified Mermaid sources to VS Code; source-mode info remains semantic-token search.
- The proof diagrams did not expose a Mermaid limitation requiring D2 support.
