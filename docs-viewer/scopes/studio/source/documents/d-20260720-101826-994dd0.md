# Processing

`processing/` is a separate Java/Processing project hosted inside this repository. It is not a website module or part of the GitHub Pages artifact.

This Studio-scope family owns how that project fits into the repository and surrounding systems. The separate [Processing documentation scope](/docs/?scope=processing&doc=d-20260714-092047-06a5db) owns concepts, artwork and algorithm notes, project architecture, and technical reference for the Processing work itself.

## Ownership Boundary

| Studio scope | Processing scope |
| --- | --- |
| repository placement and source ownership | artwork and project concepts |
| required Processing, Java, and library setup | sketch and algorithm architecture |
| development, build, test, and export lifecycle integration | implementation notes and technical reference |
| Docs Viewer scope, media, and publication integration | project-specific authoring and rendering guidance |
| links to shared repository services or checks | records of experiments and recovered knowledge |

Do not treat repository co-location as shared runtime ownership. Processing should be inspected, run, and eventually tested through its own project lifecycle.

## Current Integration

- [Processing Project Setup](/docs/?scope=studio&doc=d-20260714-164200-381988) records the current environment, adopted repository shape, recovery boundary, generated-material policy, and integration decisions.
- [Source Tree Ownership](/docs/?scope=studio&doc=d-20260524-214228-7247ef) places `processing/` alongside the repository's other independently owned roots.
- [Scope Media Management](/docs/?scope=studio&doc=d-20260720-100017-9855dd) owns cross-cutting inventory and lifecycle questions for media deliberately published through Docs Viewer.

## Code Authority

Verify exact current behavior in:

- `processing/README.md` and `processing/projects/` for the project layout and runnable sketches
- `processing/libraries/` for deliberately pinned sketchbook libraries
- `docs-viewer/config/scopes/docs_scopes.json` for the Processing documentation-scope contract
- project-owned scripts and tests as they are introduced

Keep conceptual and technical Processing documentation in the Processing scope. Add Studio children here only when the subject is repository hosting or integration with shared systems.
