# Tag Aliases

## Capability

`/analytics/tag-aliases/` maintains shorthand that resolves to canonical tags: browse/filter, import, create, edit, delete, promote an alias to a canonical tag, and demote a canonical tag into an alias.

Aliases preserve useful language without making every phrase canonical vocabulary. They can also be recorded as historical selection context on assignment rows while canonical tag IDs remain the actual assignment identity.

## Workflow

```text
load aliases + registry + assignment context
  -> edit/import or choose promote/demote
  -> validate alias key and canonical targets
  -> preview cross-file impact where required
  -> confirm
  -> server planner + atomic apply
  -> reload + activity evidence
```

Direct server writes are preferred. When the local service is unavailable, supported operations return manual patch snippets. Those snippets describe work for a maintainer; they do not become canonical state automatically.

## Durable Rules

- alias keys are slug-safe;
- an alias targets at least one canonical tag;
- one alias cannot target two tags from the same group;
- target IDs must exist in the canonical registry;
- promotion creates a canonical tag and reconciles the alias/assignment references defined by the planner;
- demotion removes a canonical identity only through an explicit replacement plan.

Limits and exact request shapes live in `tag_source_model.py` and focused tests.

## Code Map

- template/controller: `tag-aliases.html`, `tag-aliases.js`
- state/render/modal/import/save modules: `analytics-app/app/frontend/js/tag-aliases-*`
- HTTP boundary: `tag_write_api/aliases.py`, `tag_write_api/promotions.py`
- planners: `tag_alias_mutations.py`, `tag_promotion_mutations.py`
- canonical source: `analytics-app/data/canonical/tag-aliases.json`

## Weak Spots And Change Method

- Promotion/demotion spans vocabulary, aliases, and assignments, making it the highest-risk tag mutation family.
- Patch fallback can express guidance but does not share server validation, activity, atomicity, or rollback.
- Alias history in assignments is deliberately secondary metadata and can be mistaken for current resolution authority.

Start changes in the planner/model and its cross-file tests. Update route docs only for a durable workflow/ownership change, not for modal layout.
