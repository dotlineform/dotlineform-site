# Current Site Baseline Review Delivery

## Status And Purpose

This is a **proposed, write-free review delivery**. It establishes what the public catalogue currently does, which code and generated artifacts are live, and where cleanup is justified. It does not change runtime behaviour or remove files.

## Outcome

One accepted baseline covers the public catalogue from `/series/` through Work, detail, recent, search, shared shell, and failure routes. Each suspected weak spot is confirmed, narrowed, or dismissed with code, payload, route, and focused-check evidence.

The review ends with an ordered recommendation for independently complete cleanup deliveries. It must not convert every observation into work.

## Review Areas

### Routing And State

- Map every catalogue route, redirect, query parameter, local-storage value, and link producer.
- Record default, selected, restored, paginated, back-navigation, invalid, and missing-record behaviour.
- Check whether URL and persistence rules agree, especially when navigation crosses Series, Work, and detail routes.
- Identify deprecated modes, compatibility paths, duplicated URL construction, and unreachable route branches.

### CSS And Static Shell

- Map tokens, global shell, catalogue components, route-specific rules, responsive behaviour, and theme ownership across loaded stylesheets.
- Attribute selectors to current markup and modules before calling them dead.
- Compare repeated head, header, navigation, theme, footer, and script-loading markup across tracked route shells.
- Separate real drift from repetition that is currently required by the no-build static deployment model.

### Runtime Data And Generated Artifacts

- Trace the public JSON and media read by each route from fetch call to generated file family.
- Identify generated families with no current public consumer and confirm whether another tool or route owns them.
- Check route, component, navigation, and shared-helper ownership after the JavaScript refactor.
- Keep canonical catalogue source and Studio generation changes outside the review unless consumer evidence requires a separate recommendation.

### Verification

- Inventory the deploy-root validator and focused public route, runtime-module, Work-route, and theme checks.
- Record which durable data and boot boundaries are protected and which checks merely mirror implementation or presentation.
- Identify the smallest focused evidence needed for each proposed cleanup delivery.

## Likely Authority To Inspect

- `site/series/`, `site/works/`, `site/work-details/`, `site/recent/`, `site/catalogue/search/`, `site/404.html`, and the root redirect
- `site/assets/js/catalogue/routes/`, `components/`, `navigation/`, and `shared/`
- `site/assets/css/main.css` and `site/assets/css/catalogue.css`
- public catalogue JSON and media beneath `site/assets/`
- `site-tools/`, focused public-site smoke tests, and catalogue generators only where public-consumer attribution requires them

This is an inspection map, not an instruction to change every owner.

## Not In Scope

- Runtime, CSS, shell, generator, payload, or test edits.
- New routes, navigation, controls, catalogue fields, or presentation design.
- Public Docs Viewer reader internals.
- A new bundler, static-site generator, template layer, client framework, compatibility alias, or generic route abstraction.
- Removal based only on names, age, duplicate appearance, or lack of an obvious nearby import.

## Completion Evidence

- A route-and-state matrix records current catalogue entry, restoration, transition, invalid, and failure behaviour.
- A CSS and shell ownership map attributes live rules and repeated markup, with evidence for suspected dead or drifting areas.
- A consumer map connects runtime fetches to generated artifact families and flags only evidence-backed candidates for retirement.
- Existing checks are mapped to durable contracts, with focused gaps stated without adding tests during the review.
- The [Site Overview](/docs/?scope=studio&doc=d-20260720-161657-0f6696), [Public Catalogue Design](/docs/?scope=studio&doc=d-20260720-161658-42fee0), [Public Route Model](/docs/?scope=studio&doc=d-20260602-192905-260208), [Site Shell Runtime](/docs/?scope=studio&doc=d-20260331-000000-6d51e9), and [Public Catalogue Data Flow](/docs/?scope=studio&doc=d-20260331-000000-33a9c5) are checked against the evidence.
- The review recommends bounded cleanup deliveries, explicit non-goals, dependencies, validation, and stop points for user acceptance before implementation.

Do not begin runtime cleanup from this document until the review delivery itself is accepted for execution.
