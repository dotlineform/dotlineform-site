# Document Sharing UI Redesign

## Purpose

Replace the current route-shaped document-sharing UI once Docs Viewer has a canonical multiple-selection interaction. The dedicated Prepare and Returned Packages pages are useful shipped entrypoints, but their duplicated route layout should not become a long-term design constraint or receive an independent visual redesign that will be discarded.

## Proposed Direction

- Checked documents in the Docs Viewer index become the canonical input to **Prepare package**.
- The Actions command opens a compact modal for profile, format, descendant, and confirmation choices; it does not reproduce the document tree or scope selection.
- **Returned packages** opens a scope-owned package inbox hosted by Docs Viewer.
- A returned package remains one atomic, read-only record set. Its rows do not participate in index selection or partial apply.
- Existing package services, trusted metadata, scope partitioning, validation, review, and canonical handoff contracts remain in place.

## Transition

This feature is paused behind [Index Multiple Selection](/docs/?scope=studio&doc=d-20260714-151900-6fa23e). Keep the current scope-specific routes operational and make only necessary correctness, accessibility, and maintenance fixes in the meantime.

When the checked-document contract exists, replace the route shells in one bounded cutover, verify both package workflows through their new hosts, and retire the duplicate pages without compatibility aliases.

## Related Feature

[Share Document Packages](/docs/?scope=studio&doc=d-20260718-155350-c84e62) remains the shipped authority for document-package behavior and service ownership. This feature changes only its browser composition.
