# Document Sharing UI Redesign

## Purpose

Replace the former route-shaped document-sharing UI now that Docs Viewer has a canonical multiple-selection interaction. Prepare no longer duplicates selection in a dedicated page. Returned packages remains a scope-owned whole-package workflow rather than a selection consumer, but it no longer needs a separate browser route.

## Shipped Direction

- Checked documents in the Docs Viewer index become the canonical input to **Prepare package**.
- The Actions command opens a compact modal for profile, format, descendant, and confirmation choices; it does not reproduce the document tree or scope selection.
- **Review package** opens a scope-owned modal hosted by Docs Viewer and prepares one complete returned package for Docs Review.
- **Returned packages is not a selection consumer.** A returned package remains one whole, atomic, read-only record set. Its rows do not participate in index selection, filtering, or partial apply.
- Existing package services, trusted metadata, scope partitioning, validation, review, and canonical handoff contracts remain in place.

[Prepare Package UI Redesign Delivery](/docs/?scope=studio&doc=d-20260722-133457-e90e21) is complete. The shipped [Prepare Package](/docs/?scope=studio&doc=d-20260722-151224-7b61c4) user workflow records its selection rules and action calls.

[Review Package UI Redesign Delivery](/docs/?scope=studio&doc=d-20260722-204025-15ea54) completed the scope-owned review modal, complete Docs Viewer Import boundary, separate summary/hierarchy retirement, old route removal, final verification, and durable documentation.

## Transition

The checkbox-selection dependency plus the **Prepare package** and **Review package** workflows are shipped. Their compact manage Actions own both operations, and both duplicate package routes are retired without aliases.

Each delivery retired its route only after the corresponding Action worked end to end. Neither cutover adds a compatibility alias.

This redesign is complete. Future multi-select consumers or wider Docs Viewer styling remain separate features.

## Related Feature

[Share Document Packages](/docs/?scope=studio&doc=d-20260718-155350-c84e62) remains the shipped authority for document-package behavior and service ownership. This feature changes only its browser composition.
