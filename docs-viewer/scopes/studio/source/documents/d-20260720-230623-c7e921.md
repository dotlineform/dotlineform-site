# Document Sharing UI Redesign

## Purpose

Replace the current route-shaped document-sharing UI once Docs Viewer has a canonical multiple-selection interaction. The dedicated Prepare and Returned Packages pages are useful shipped entrypoints, but their duplicated route layout should not become a long-term design constraint or receive an independent visual redesign that will be discarded.

## Proposed Direction

- Checked documents in the Docs Viewer index become the canonical input to **Prepare package**.
- The Actions command opens a compact modal for profile, format, descendant, and confirmation choices; it does not reproduce the document tree or scope selection.
- **Returned packages** opens a scope-owned package inbox hosted by Docs Viewer.
- **Returned packages is not a selection consumer.** A returned package remains one whole, atomic, read-only record set. Its rows do not participate in index selection, filtering, or partial apply.
- Existing package services, trusted metadata, scope partitioning, validation, review, and canonical handoff contracts remain in place.

## Delivery

[Prepare Package UI Redesign Delivery](/docs/?scope=studio&doc=d-20260722-133457-e90e21) is the active first slice. Its selection-only Action registration is complete; the compact workflow and dedicated-route retirement remain gated. Returned Packages remains a separate later slice.

## Transition

The checkbox-selection dependency is shipped, and **Prepare package** is the first active delivery. Keep the current Prepare route operational until the compact Action workflow owns the complete operation; keep the Returned Packages route unchanged and make only necessary correctness, accessibility, and maintenance fixes there.

The Prepare delivery retires only its route after the selection Action works end to end. The Returned Packages route remains operational until its own later delivery. Neither cutover adds compatibility aliases.

## Related Feature

[Share Document Packages](/docs/?scope=studio&doc=d-20260718-155350-c84e62) remains the shipped authority for document-package behavior and service ownership. This feature changes only its browser composition.
