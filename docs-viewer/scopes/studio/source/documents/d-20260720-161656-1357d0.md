# Site

`site/` is the tracked public website and GitHub Pages deploy root. The public catalogue is centred on `/series/`, the Works home reached from `/`.

Public Docs Viewer routes share parts of the site shell but remain documented under [Docs Viewer](/docs/?scope=studio&doc=d-20260424-000000-50b63f). This section owns the public catalogue experience and the site-wide shell around it.

## Understand The Site

- [Site Overview](/docs/?scope=studio&doc=d-20260720-161657-0f6696) — product surface, execution path, ownership, extension method, and weak spots.
- [Public Catalogue Design](/docs/?scope=studio&doc=d-20260720-161658-42fee0) — navigation, presentation, interaction, responsive behaviour, and frontend ownership.
- [Public Route Model](/docs/?scope=studio&doc=d-20260602-192905-260208) — stable catalogue URLs and query-state rules.
- [Site Shell Runtime](/docs/?scope=studio&doc=d-20260331-000000-6d51e9) — shared head, header, navigation, theme, footer, and asset-loading boundary.
- [Public Catalogue Data Flow](/docs/?scope=studio&doc=d-20260331-000000-33a9c5) — runtime payloads read by each catalogue route.

## Change Site Content

- [Studio](/docs/?scope=studio&doc=d-20260423-000000-d015e6) owns catalogue authoring and maintenance.
- [Catalogue Architecture](/docs/?scope=studio&doc=d-20260401-000000-a11bf3) owns the source-to-public-projection boundary.
- [Catalogue Search](/docs/?scope=studio&doc=d-20260602-160839-4ebe27) owns the public catalogue search surface and index.

The public browser never writes catalogue source. Studio validates and changes canonical records, then its catalogue services rebuild the public projections consumed by Site.

## Operate The Site

- [Public Site Preview](/docs/?scope=studio&doc=d-20260523-203756-b41c00) — serve the checked deploy tree locally and run deploy-root validation.
- [Site Consistency Audit](/docs/?scope=studio&doc=d-20260331-000000-cdc571) — check catalogue projections, routes, links, and media relationships.
- [Testing](/docs/?scope=studio&doc=d-20260501-174746-efd581) — choose focused runtime, browser-boundary, or visual evidence.
- [GitHub Actions](/docs/?scope=studio&doc=d-20260612-153500-7eadc8) — public-site validation and deployment workflow.

## Code Authority

- `site/series/`, `site/works/`, `site/work-details/`, `site/recent/`, and `site/catalogue/search/` for tracked route shells
- `site/assets/js/catalogue/` for public catalogue routes, components, navigation, and shared helpers
- `site/assets/css/main.css` and `site/assets/css/catalogue.css` for the current presentation
- `site-tools/` for deploy-root validation and static preview tooling

Use code, config, generated payloads, and focused tests for exact current fields and behaviour. These documents provide the stable map.
