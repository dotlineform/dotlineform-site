# Docs Viewer Management Script Overview

This is the ownership map for the scripts and service modules behind Docs Viewer management. Endpoint behavior is documented separately in [Endpoint Overview](/docs/?scope=studio&doc=d-20260607-222033-647b52).

## Runtime And Dispatch

- [Service Entrypoints](/docs/?scope=studio&doc=d-20260607-222033-ccc471): `docs-viewer/bin/docs-viewer`, `docs-viewer/services/docs_viewer_service.py`
- [Route Dispatch Scripts](/docs/?scope=studio&doc=d-20260607-222033-71d1be): `docs_management_routes.py`, `docs_management_service.py`, `docs_management_context.py`

## Read And Config Helpers

- [Read And Config Scripts](/docs/?scope=studio&doc=d-20260607-222033-f2a546): generated reads, capabilities, source-config report, source-config settings, and source model helpers

## Write Workflows

- [Source Mutation Scripts](/docs/?scope=studio&doc=d-20260607-222033-c0b57a): create, metadata, viewability, move, delete, and scope lifecycle apply helpers
- [Import Scripts](/docs/?scope=studio&doc=d-20260607-222033-c902f6): staged source import service and format conversion helpers
- [Source Editor Scripts](/docs/?scope=studio&doc=d-20260607-222033-704641): source body read/rebuild and local editor open helpers
- [Rebuild Follow-Through Scripts](/docs/?scope=studio&doc=d-20260607-222033-dd2265): builder command orchestration and live-watcher suppression helpers
- [Audit Scripts](/docs/?scope=studio&doc=d-20260607-222033-6331b3): broken-link route adapter and CLI audit engine

## Ownership Rule

Endpoint modules should parse and dispatch HTTP-shaped input. Workflow modules should own validation, planning, source writes, generated-output rebuilds, and activity/log side effects. Builders should own generated artifact formats.
