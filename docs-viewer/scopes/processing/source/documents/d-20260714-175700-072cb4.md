## Overview

This brief turns the conceptual question—“Are images alive?”—into practical modules for a Processing-based application. The goal is to operationalise *punctum* (Barthes), *aura* (Benjamin), cultural *ways of seeing* (Berger), and perceptual salience (pareidolia/face systems) into adjustable, testable image-composition behaviours [Barthes1981; Benjamin1936; Berger1972; Wardle2020; Akdeniz2018; Rossion2024].

1. **Punctum Mapper** — interactive hotspots that the user marks as “alive”; drives weighted blending and local contrast.
2. **Face/Salience Bias** — heuristic or ML-assisted detection to weight faces/eye-like patterns.
3. **Aura Slider** — balances “singular presence” vs. “reproductive remix” via source-count, compression, and artifact parameters.
4. **Relational Mode** — render parameters adapt to viewer interaction (dwell time, cursor distance, sequence history).
5. **Agency Log** — track how outputs influence subsequent choices, treating images as agents within the workflow.

## Algorithms & Heuristics

### 1) Punctum Weight Field

Create a spatial weight map from user-marked puncta; combine Gaussians: `w(x,y) = Σ_i (w_i * exp(-d^2 / (2σ_i^2)))`. Use this to boost local contrast (unsharp mask) and increase layer opacity.

### 2) Face/Salience Bias

Option A: Lightweight heuristics for “eye-like” patterns (two dark blobs + lighter surrounding). Option B (if allowed): integrate a Java face detector library for Processing; otherwise prepare for external precomputed face-masks. Empirical grounding: illusory faces recruit face-selective cortex (FFA/OFA), explaining why such regions feel “alive” [Wardle2020; Akdeniz2018; Rossion2024].

### 3) Aura Slider

- **Low aura (0→0.3):** Prefer single-source dominance, minimal processing, gentle contrast, fewer modes (e.g., `BLEND`).
- **Mid aura (0.3→0.7):** 2–3 sources, tasteful modes (`MULTIPLY`/`SCREEN`), subtle grain.
- **High aura (0.7→1.0):** Multi-source remix, visible texture, halation/glow, vignetting allowed.

This parameter performs Benjamin’s idea of aura vs. reproducibility as a continuous control [Benjamin1936].

### 4) Relational Mode

Map viewer interaction to rendering: `dwellTime → local contrast`, `cursorDistance → vignette strength`, `revisitCount → color warmth`. This embodies co-arising perception from Buddhist relational ontology [PratityaOxford; SunyataLion].

### 5) Agency Log

Append JSON lines after each render with parameter snapshots and thumbnails. Use these logs to inform next-run defaults (images “influencing” workflow). Concept from anthropological “art as agent” [Gell1998].

## Processing Implementation Tips

- **UHD safety:** Use `PGraphics` for compositing; avoid per-pixel loops where possible; batch-save with `beginRecord()` disabled for speed.
- **Spatial blending:** For per-pixel weighting, chunk processing in tiles (e.g., 256×256) to manage memory; reuse `pixels[]` buffers.
- **Auto Levels:** Use percentile clipping (e.g., 0.5–1%) computed once per layer unless relation-driven refresh is required.
- **Face-mask fallback:** Let users import binary masks (white = face) to drive salience without ML dependencies.
- **Batch mode:** Run in `setup()` only; no `draw()`; iterate source sets and exit.
