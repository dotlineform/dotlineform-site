# Processing Blob Studio for Web Gallery Shapes

## Purpose

This guide describes an offline Processing tool for creating reusable organic SVG shapes for the website gallery.

The tool lives inside the existing website repository:

```text
dotlineform-site/
├── processing/
│   └── blob-studio/
├── assets/
├── docs/
├── _works/
└── ...
```

Processing is used only as an authoring and generation environment.

The website runtime remains standard:

- HTML;
- CSS;
- SVG;
- JavaScript;
- JSON.

The Processing tool generates clean SVG paths and shape metadata that the site can use for:

- clipping individual thumbnails;
- defining circular, elliptical or organic gallery boundaries;
- testing packed thumbnail positions;
- creating repeatable families of related forms;
- exporting deterministic shapes from saved seeds and parameters.

## 2. Processing’s role

Processing should be responsible for:

- seeded procedural shape generation;
- interactive parameter controls;
- manual point and handle editing;
- previewing the shape with a real artwork thumbnail;
- previewing many repeated shapes;
- testing a form as a gallery boundary;
- exporting SVG;
- exporting normalised path data;
- saving editable presets;
- generating or updating a site shape registry.

Processing should not be required by the deployed website.

The workflow is:

```text
Processing Blob Studio
        ↓
SVG + JSON preset
        ↓
shape registry
        ↓
website build/runtime
        ↓
CSS/SVG clipping and JavaScript layout
```

## 5. Shape data model

A blob can be represented as a sequence of anchors and Bézier controls.

```java
class BlobSegment {
  PVector anchor;
  PVector controlIn;
  PVector controlOut;

  BlobSegment(
    PVector anchor,
    PVector controlIn,
    PVector controlOut
  ) {
    this.anchor = anchor;
    this.controlIn = controlIn;
    this.controlOut = controlOut;
  }
}
```

The complete form stores its generator parameters and path segments.

```java
class Blob {
  String shapeId;

  int seed;
  int pointCount;

  float irregularity;
  float smoothness;
  float scaleX;
  float scaleY;
  float rotationDeg;

  ArrayList<BlobSegment> segments;

  Blob() {
    segments = new ArrayList<BlobSegment>();
  }
}
```

This data structure should drive:

- the on-screen preview;
- Processing SVG export;
- SVG path-string export;
- JSON preset export;
- normalised website paths.

## 6. Seeded generation

A deterministic seed ensures that a shape can be regenerated exactly.

```java
void generateBlob(Blob blob) {
  randomSeed(blob.seed);
  noiseSeed(blob.seed);

  blob.segments.clear();

  ArrayList<PVector> anchors =
    createRadialAnchors(
      blob.pointCount,
      width * 0.5,
      height * 0.5,
      min(width, height) * 0.38,
      blob.irregularity
    );

  createBezierSegments(
    blob,
    anchors,
    blob.smoothness
  );
}
```

A basic radial anchor generator:

```java
ArrayList<PVector> createRadialAnchors(
  int count,
  float centreX,
  float centreY,
  float baseRadius,
  float irregularity
) {
  ArrayList<PVector> points =
    new ArrayList<PVector>();

  for (int index = 0; index < count; index++) {
    float angle =
      TWO_PI * index / count;

    float variation =
      1.0 +
      random(
        -irregularity,
        irregularity
      );

    float radius =
      baseRadius * variation;

    float x =
      centreX +
      cos(angle) * radius;

    float y =
      centreY +
      sin(angle) * radius;

    points.add(
      new PVector(x, y)
    );
  }

  return points;
}
```

A more refined version could combine:

- seeded random values;
- Perlin noise;
- radial constraints;
- minimum point spacing;
- concavity limits;
- symmetry controls.

## 7. Bézier control generation

A smooth closed path can be constructed by calculating tangents around each anchor.

```java
void createBezierSegments(
  Blob blob,
  ArrayList<PVector> anchors,
  float smoothness
) {
  int count = anchors.size();

  for (int index = 0; index < count; index++) {
    PVector previous =
      anchors.get(
        (index - 1 + count) % count
      );

    PVector current =
      anchors.get(index);

    PVector next =
      anchors.get(
        (index + 1) % count
      );

    PVector tangent =
      PVector.sub(next, previous);

    tangent.normalize();

    float previousDistance =
      PVector.dist(current, previous);

    float nextDistance =
      PVector.dist(current, next);

    float controlLength =
      min(
        previousDistance,
        nextDistance
      ) *
      smoothness *
      0.5;

    PVector controlIn =
      PVector.sub(
        current,
        PVector.mult(
          tangent,
          controlLength
        )
      );

    PVector controlOut =
      PVector.add(
        current,
        PVector.mult(
          tangent,
          controlLength
        )
      );

    blob.segments.add(
      new BlobSegment(
        current.copy(),
        controlIn,
        controlOut
      )
    );
  }
}
```

This provides a usable baseline. Manual handle editing can later override the generated controls.

## 8. Drawing the blob in Processing

```java
void drawBlob(Blob blob) {
  if (blob.segments.isEmpty()) {
    return;
  }

  pushMatrix();

  translate(
    width * 0.5,
    height * 0.5
  );

  rotate(
    radians(blob.rotationDeg)
  );

  scale(
    blob.scaleX,
    blob.scaleY
  );

  translate(
    -width * 0.5,
    -height * 0.5
  );

  beginShape();

  BlobSegment first =
    blob.segments.get(0);

  vertex(
    first.anchor.x,
    first.anchor.y
  );

  for (
    int index = 1;
    index <= blob.segments.size();
    index++
  ) {
    BlobSegment previous =
      blob.segments.get(
        (index - 1) %
        blob.segments.size()
      );

    BlobSegment current =
      blob.segments.get(
        index %
        blob.segments.size()
      );

    bezierVertex(
      previous.controlOut.x,
      previous.controlOut.y,

      current.controlIn.x,
      current.controlIn.y,

      current.anchor.x,
      current.anchor.y
    );
  }

  endShape(CLOSE);

  popMatrix();
}
```

The same segment structure should be used for both display and export.

## 9. Previewing a thumbnail inside the blob

A real site thumbnail can be loaded into Processing:

```java
PImage previewImage;

void setup() {
  size(900, 900);

  previewImage =
    loadImage("preview-image.webp");
}
```

There are several ways to preview the crop.

A simple first implementation is:

1. render the image into an off-screen `PGraphics`;
2. render the blob into a mask buffer;
3. apply the mask;
4. draw the result.

```java
PImage createMaskedPreview(
  PImage source,
  Blob blob
) {
  PGraphics imageBuffer =
    createGraphics(width, height);

  PGraphics maskBuffer =
    createGraphics(width, height);

  imageBuffer.beginDraw();
  imageBuffer.background(0, 0);
  imageBuffer.imageMode(CENTER);
  imageBuffer.image(
    source,
    width * 0.5,
    height * 0.5,
    width,
    height
  );
  imageBuffer.endDraw();

  maskBuffer.beginDraw();
  maskBuffer.background(0);
  maskBuffer.fill(255);
  maskBuffer.noStroke();
  drawBlobInto(maskBuffer, blob);
  maskBuffer.endDraw();

  PImage result =
    imageBuffer.get();

  result.mask(
    maskBuffer.get()
  );

  return result;
}
```

The implementation details can be refined by Codex, but the important point is that Blob Studio should preview actual artwork rather than only solid black shapes.

## 10. SVG export with Processing

Processing’s SVG renderer can export the visible vector form.

```java

```

```java
void exportProcessingSvg(
  Blob blob,
  String filename
) {
  beginRecord(
    SVG,
    filename
  );

  background(255);
  noStroke();
  fill(0);

  drawBlob(blob);

  endRecord();
}
```

Example:

```java
exportProcessingSvg(

  "exports/svg/blob-001.svg"

```

This is useful as a visual master and for opening in Inkscape or another SVG editor.

However, the website should preferably use a deliberately generated clean path string rather than depending on the exact structure of Processing’s complete SVG output.

## 11. Direct SVG path generation

A path string can be generated directly from the segment data.

```java
String toSvgPath(Blob blob) {
  if (blob.segments.isEmpty()) {
    return "";
  }

  StringBuilder path =
    new StringBuilder();

  BlobSegment first =
    blob.segments.get(0);

  path.append(
    "M " +
    formatNumber(first.anchor.x) +
    " " +
    formatNumber(first.anchor.y)
  );

  for (
    int index = 1;
    index <= blob.segments.size();
    index++
  ) {
    BlobSegment previous =
      blob.segments.get(
        (index - 1) %
        blob.segments.size()
      );

    BlobSegment current =
      blob.segments.get(
        index %
        blob.segments.size()
      );

    path.append(
      " C " +
      formatNumber(previous.controlOut.x) +
      " " +
      formatNumber(previous.controlOut.y) +
      ", " +
      formatNumber(current.controlIn.x) +
      " " +
      formatNumber(current.controlIn.y) +
      ", " +
      formatNumber(current.anchor.x) +
      " " +
      formatNumber(current.anchor.y)
    );
  }

  path.append(" Z");

  return path.toString();
}
```

```java
String formatNumber(float value) {
  return nf(value, 0, 3);
}
```

## 12. Normalised website path

The site should use coordinates ranging from `0` to `1`.

This allows the SVG clipping path to use:

```svg
clipPathUnits="objectBoundingBox"
```

Normalisation requires a known source coordinate space.

For example, if Blob Studio uses a `1000 × 1000` export canvas:

```java
final float EXPORT_SIZE =
  1000.0;
```

```java
float normaliseCoordinate(float value) {
  return value / EXPORT_SIZE;
}
```

A normalised path writer:

```java
String toNormalisedSvgPath(Blob blob) {
  if (blob.segments.isEmpty()) {
    return "";
  }

  StringBuilder path =
    new StringBuilder();

  BlobSegment first =
    blob.segments.get(0);

  path.append(
    "M " +
    formatNormalised(first.anchor.x) +
    " " +
    formatNormalised(first.anchor.y)
  );

  for (
    int index = 1;
    index <= blob.segments.size();
    index++
  ) {
    BlobSegment previous =
      blob.segments.get(
        (index - 1) %
        blob.segments.size()
      );

    BlobSegment current =
      blob.segments.get(
        index %
        blob.segments.size()
      );

    path.append(
      " C " +
      formatNormalised(
        previous.controlOut.x
      ) +
      " " +
      formatNormalised(
        previous.controlOut.y
      ) +
      ", " +
      formatNormalised(
        current.controlIn.x
      ) +
      " " +
      formatNormalised(
        current.controlIn.y
      ) +
      ", " +
      formatNormalised(
        current.anchor.x
      ) +
      " " +
      formatNormalised(
        current.anchor.y
      )
    );
  }

  path.append(" Z");

  return path.toString();
}
```

```java
String formatNormalised(float value) {
  return nf(
    value / EXPORT_SIZE,
    0,
    6
  );
}
```

A typical result:

```svg
M 0.500000 0.030000
C 0.780000 0.050000,
  0.960000 0.270000,
  0.920000 0.570000
...
Z
```

## 16. Site shape registry

The website should not need to load many unrelated SVG files individually.

Instead, selected shapes can be collected into a registry:

```json
{
  "version": 1,
  "generated_at_utc": "2026-07-20T12:00:00Z",
  "shapes": [
    {
      "shape_id": "blob-001",
      "label": "Rounded asymmetric",
      "type": "blob",
      "normalised_path": "M 0.500000 0.030000 C ... Z"
    },
    {
      "shape_id": "leaf-001",
      "label": "Broad leaf",
      "type": "leaf",
      "normalised_path": "M 0.050000 0.550000 C ... Z"
    }
  ]
}
```

Suggested site location:

```text
assets/shapes/shape-registry.json
```

The registry could be updated by:

- Blob Studio directly;
- a small Python build script;
- the existing site build pipeline;
- a dedicated command run after exporting shapes.

For reliability, a build script that reads selected presets and generates the public registry is preferable to letting Blob Studio rewrite site files unpredictably.

## 19. Shape assignment

Shapes can be assigned in several ways.

### Repeating sequence

```js
const shapeIds = [
  "blob-001",
  "blob-002",
  "leaf-001",
  "droplet-001",
];

items.forEach((item, index) => {
  item.dataset.shape =
    shapeIds[index % shapeIds.length];
});
```

### Stable assignment by work ID

```js
function hashString(value) {
  let hash = 0;

  for (
    let index = 0;
    index < value.length;
    index++
  ) {
    hash =
      ((hash << 5) - hash) +
      value.charCodeAt(index);

    hash |= 0;
  }

  return Math.abs(hash);
}
```

```js
function assignStableShape(
  item,
  shapeIds
) {
  const workId =
    item.dataset.workId ?? "";

  const shapeIndex =
    hashString(workId) %
    shapeIds.length;

  item.dataset.shape =
    shapeIds[shapeIndex];
}
```

This produces repeatable variation between visits.

## 20. Shapes as gallery boundaries

A generated shape can also define the whole gallery boundary:

```css
.gallery[data-boundary="blob-001"] {
  clip-path: url("#blob-001");
}
```

However, visually clipping the gallery is not sufficient if thumbnails must remain fully inside the boundary.

The JavaScript packing algorithm should use the same path to test candidate positions.

This gives Blob Studio an additional useful preview mode:

- show the shape as the outer boundary;
- generate candidate thumbnail positions;
- mark valid and invalid positions;
- estimate capacity;
- compare different thumbnail sizes.

## 21. Exporting geometry for packing

The website may need the shape in a larger coordinate space for point testing.

The JSON preset can include both:

- a normalised `0–1` path for CSS/SVG clipping;
- a `0–1000` path for geometry and packing.

```json
{
  "shape_id": "blob-001",
  "view_box": [0, 0, 1000, 1000],
  "svg_path": "M 500 30 C ... Z",
  "normalised_path": "M 0.5 0.03 C ... Z"
}
```

The site can create an invisible SVG path and use:

```js
path.isPointInFill(point)
```

Candidate thumbnail positions can be tested against the same shape used visually.

## 22. Packing preview in Processing

Blob Studio could display a hexagonal candidate lattice inside the current shape.

A basic candidate generator:

```java
ArrayList<PVector> generateHexCandidates(
  float areaWidth,
  float areaHeight,
  float itemDiameter,
  float gap
) {
  ArrayList<PVector> positions =
    new ArrayList<PVector>();

  float spacingX =
    itemDiameter + gap;

  float spacingY =
    spacingX * sqrt(3) / 2;

  int rows =
    ceil(areaHeight / spacingY);

  int columns =
    ceil(areaWidth / spacingX);

  for (int row = 0; row <= rows; row++) {
    float y =
      row * spacingY;

    float offsetX =
      row % 2 == 1
        ? spacingX * 0.5
        : 0;

    for (
      int column = 0;
      column <= columns;
      column++
    ) {
      float x =
        column * spacingX +
        offsetX;

      positions.add(
        new PVector(x, y)
      );
    }
  }

  return positions;
}
```

Testing full circular thumbnails against a Bézier boundary is more involved.

Possible approaches include:

- drawing the boundary into a mask buffer;
- testing the centre and circumference samples against the mask;
- using a flattened polygon approximation;
- exporting the path and relying on the browser for final geometry testing.

For Blob Studio, a raster mask-based preview may be sufficient.

## 23. Batch generation

Processing is particularly useful for creating related families of shapes.

```java
void generateBatch(
  int count,
  int startingSeed
) {
  for (
    int index = 0;
    index < count;
    index++
  ) {
    Blob blob =
      createBlobFromSeed(
        startingSeed + index
      );

    blob.shapeId =
      "blob-" +
      nf(index + 1, 3);

    saveStandaloneSvg(
      blob,
      "exports/svg/" +
      blob.shapeId +
      ".svg"
    );

    saveBlobPreset(
      blob,
      "exports/json/" +
      blob.shapeId +
      ".json"
    );
  }
}
```

A batch preview should make it possible to:

- review all generated forms;
- mark favourites;
- reject weak shapes;
- rename selected forms;
- export only curated shapes;
- create a coherent visual family.

## 24. Curated versus fully random output

The website should probably use a curated shape set rather than generating new contours at runtime.

Recommended model:

```text
Processing generates many candidates
        ↓
selected shapes are curated
        ↓
curated shapes enter the site registry
        ↓
website assigns them deterministically
```

This gives:

- visual quality control;
- stable output;
- predictable clipping;
- fewer browser calculations;
- reproducible layouts;
- easier testing.

The randomness belongs primarily in the authoring process, not necessarily in the deployed site.

The stable ID should be the canonical reference.

## 30. Validation

Before a shape enters the site registry, validate that:

- the path is closed;
- the path stays within the expected view box;
- the normalised path remains approximately within `0–1`;
- the shape has no unintended self-intersections;
- the shape has sufficient visible area;
- the form is not excessively narrow;
- a typical image crop remains readable;
- repeated forms do not create excessive visual noise;
- the shape works at small thumbnail sizes;
- the path renders consistently in supported browsers.

Blob Studio could display validation warnings before export.

## 32. Recommended architectural separation

Keep these parts distinct:

```text
BlobGenerator
    creates initial anchors

BlobEditor
    modifies anchors and handles

Blob
    stores canonical geometry

BlobRenderer
    draws previews

SvgPathWriter
    creates clean SVG paths

BlobPreset
    saves and loads editable settings

ShapeRegistryWriter
    generates website registry data
```

This prevents the GUI, generation logic and export format from becoming tightly coupled.

## 33. Practical recommendation

For the current gallery project:

1. Keep Blob Studio under the existing repo’s `processing/` folder.
2. Use Processing for generation, curation and preview.
3. Save editable JSON presets as the canonical authoring source.
4. Export clean standalone SVG files for inspection and editing.
5. Export normalised SVG path data for the website.
6. Build a small selected shape registry.
7. Let the website use only CSS, SVG and JavaScript.
8. Calculate complex gallery layouts lazily and cache them.
9. Keep runtime shape assignment deterministic.
10. Treat the Processing tool as a reusable site asset generator, not part of the deployed application.

This approach fits the existing repository structure while keeping the public site lightweight and standards-based.
