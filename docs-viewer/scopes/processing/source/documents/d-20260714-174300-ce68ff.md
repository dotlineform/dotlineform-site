# Redesigning I Ching symbols


```
I-ching is composed of hexagrams which are straight lines. How might these symbols be represented differently, for example, as curves or interlocking shapes? I want to create a different graphical representation whilst retaining the logic and structure of the hexagram system of symbols 

```


That’s a brilliant idea — to reinterpret the **I Ching’s hexagrams through curves, interlocking shapes, or organic geometries while preserving their internal logic (yin/yang, binary layering, and relational symmetry).**
## **Let’s unpack this carefully so that you can design a visual system that is both faithful and newly expressive.**

⸻
## 
**1. The Original Logic of the Hexagram**

## **Each hexagram is a stack of six lines, each either:**
	•	**Yang (solid line — “⚊”)**
	•	**Yin (broken line — “⚋”)**

Thus:
	•	Each hexagram encodes a **6-bit binary pattern** (2⁶ = 64 possibilities).
	•	The lower trigram (3 lines) and upper trigram (3 lines) can each be read independently, so each hexagram = (trigram × trigram).
	•	Transitions between hexagrams (via line change) express **change and transformation**.

This binary and hierarchical logic should remain intact in any new visual system.

⸻
## 
**2. Re-imagining the Line as a Curve or Shape**

Here are several visual translation strategies:

**Traditional Element**	**Conceptual Essence**	**Possible Curved / Interlocking Equivalent**
**Yang line (⚊)**	Unbroken, active, outward flow	**Continuous curve**, **convex arc**, **rising wave**, **closed ring**, or **bold stroke**
**Yin line (⚋)**	Divided, receptive, inward flow	**Interrupted curve**, **concave arc**, **open circle**, **gap**, **dashed stroke**, or **two mirrored curves**
**Trigram (3 lines)**	Compound of forces (Heaven, Earth, etc.)	**Nested trio of arcs**, **three lobes of a petal**, **tri-part spiral**, **three-fold knot**
**Hexagram (6 lines)**	Interaction of two trigrams (macro-micro)	**Two interlocking motifs** — upper and lower forms joined at a seam or mirrored axis


⸻
## 
**3. Structural Translation Framework**

Think of a **mapping function from line position → geometric parameter:**

**Line Position (bottom = 1)**	**Role**	**Visual Variable**
1–2 (Lower Trigram)	Foundation / Earth	**Radius, curvature, thickness**
3–4 (Middle)	Human / Mediation	**Phase or orientation**
5–6 (Upper Trigram)	Heaven / Potential	**Opacity, colour hue, or outward expansion**

## **Thus each hexagram becomes a parametric glyph, not just stacked bars.**
## 
## **For instance:**
	•	Yang = curvature = +1, Yin = curvature = –1
	•	Map line index to radius → generate 6 concentric arcs
	•	Alternate filled and open segments → organic mandala

⸻
## 
**4. Possible Design Families**
## 
**A. Waveform Hexagrams**
## 
## **Each line → sinusoidal curve:**
	•	Yang = full amplitude
	•	Yin = half amplitude, or phase-shifted
Stack vertically or wrap around a circle to create continuous wave bands.
→ Visual resemblance to **sound / energy** fields.

**B. Circular or Radial Hexagrams**
## 
## **Unfold the six lines around a circle:**
	•	Yin/Yang become alternating **arcs** (present/absent)
	•	Lower trigram = inner ring; upper trigram = outer ring
→ Each hexagram = unique **two-ring pattern**, like a fingerprint.

**C. Interlocking Petals / Knots**

Represent Yin as **open, Yang as closed, then use six linked petals or loops.**
## **→ Produces organic, mandala-like geometry reminiscent of Celtic knots or DNA helices.**
## 
**D. Vector-Field Interpretation**

## **Treat each line as a vector or flow direction:**
	•	Yang = outward pointing arrow
	•	Yin = inward curve
Use a circular field where each line defines a vector angle → the full hexagram = a field configuration.

⸻
## 
**5. Maintaining Logical Mappings**

You could encode values numerically (binary → 0/1 → curve parameter). Example:


```
0 (yin) = curve bends left

```

 <span style="font-family: .AppleSystemUIFontMonospaced-Regular; font-size: 15.0;">
     1 (yang) = curve bends right
 </span>

## **For hexagram ䷀ (Qian = 111111): all curves bend right → perfect symmetry.**
## **For hexagram ䷁ (Kun = 000000): all curves bend left → opposite polarity.**
## **Intermediate hexagrams show alternating curvature.**
## 
## **This preserves the computational logic while changing the visual vocabulary.**

⸻
## 
**6. Dynamic or Transformational Representation**

## **Because I Ching is about change, animate transitions:**
	•	Lines **morph** from straight to curved.
	•	Yin gaps **close** and **open** dynamically.
	•	Two hexagrams’ difference → “delta pattern” drawn as **interference between curves**.

⸻
## 
**7. Inspirations & Analogues**

**Inspiration Source**	**Relevance**
**Yijing calligraphy**	Already emphasizes brush-curve energy in the line; use this as stylistic base.
**Topological diagrams**	Curved continuity while maintaining discrete structure.
**Yin–Yang taijitu**	Circular logic of duality mapped to hexagram pairs.
**Genetic code helix diagrams**	Parallel six-bit binary to codon triplets.
**Neural network or circuit schematics**	Shows connections between layers (trigrams).


⸻
## 
**8. Example Concept**
## 
**“Curved Hexagram Mandala”**
	•	6 concentric circles.
	•	Each circle divided into 2 segments (Yin = gap; Yang = filled).
	•	Lower trigram = inner 3 circles, upper = outer 3.
	•	Use colours or curvature to express element (Heaven, Earth, Water, Fire…).

Each hexagram becomes a **unique radial signature**, readable but visually organic.

⸻
## 
**9. Implementation Notes**

## **You can prototype this in Processing or p5.js:**
	•	Input: six-bit binary array [1,0,1,1,0,0]
	•	For each bit → draw curved segment, ring, or Bézier
	•	Use mapping functions for curvature, opacity, hue
→ Output static glyph or morph animation between states.

⸻
## 
**10. Philosophical Echo**

Curves re-introduce the sense of **flow and impermanence, aligning more closely with the I Ching’s original meaning as “Book of Change.”
Straight lines symbolise order; curves symbolise transformation — together, they mirror how structure and flux coexist.**

Short answer: the curves aren’t “linear in x” (i.e., not straight lines), but they’re **linear combinations** of basis functions. That means you can write each curve in several equivalent **linear** forms—use whatever is most convenient for math or code.

**Download — all forms in one tidy brief:**
[I Ching — Superposition Curves: Linear Forms](sandbox:/mnt/data/IChing_Superposition_Linear_Forms_2025-10-21.html?_chatgptios_conversationID=68f7a38c-826c-8325-9e46-ee850ef13373&_chatgptios_messageID=53ba5d15-55d3-4e27-a75a-d9f3819bb938)

**The main forms (summary)**
	1.	**Amplitude–phase cosine sum (working form)**
Y(x)=\sum_{i=1}^{6} w_i\cos(2\pi f_0 i x+\phi_i). Here w_i are weights (e.g., 1/i), f_0 is the base frequency, and \phi_i\in\{0,\pi\} encodes yin/yang. (Superposition idea.)
Refs: Britannica (superposition); Wikipedia (Fourier series).
	2.	**Cos–sin coefficient form (phase-free)**
Y(x)=\sum_i\big[C_i\cos(\omega_i x)+S_i\sin(\omega_i x)\big] with \omega_i=2\pi f_0 i, C_i=w_i\cos\phi_i, S_i=-w_i\sin\phi_i.
For \phi_i\in\{0,\pi\}: S_i=0, C_i=\pm w_i.
Refs: Wikipedia (Fourier series).
	3.	**Complex exponential (phasor)**
Y(x)=\Re\{\sum_i A_i\,e^{j\omega_i x}\}, A_i=w_i e^{j\phi_i}.
Refs: Wikipedia (Euler’s formula).
	4.	**Matrix / regression form (sampled points)**
y = X\beta with columns of X as \cos(\omega_i x_k),\sin(\omega_i x_k); \beta=[C_1,S_1,\dots,C_6,S_6]^T.
Refs: Britannica (Fourier analysis).
	5.	**Chebyshev / polynomial-in-**\cos\theta
Let \theta=2\pi f_0 x. Then \cos(i\theta)=T_i(\cos\theta) → Y(x)=\sum_i a_i\,T_i(\cos\theta).
Refs: Wikipedia (Chebyshev polynomials).
	6.	**Frequency-domain (Fourier transform) distribution**
\hat Y(\omega) is a sum of delta spikes at \pm\omega_i with coefficients from (C_i,S_i).
Refs: Wikipedia (Fourier series/transform), Britannica (Fourier analysis).
	7.	**Linear differential equation satisfied by the curve**
Each component solves y’’+\omega_i^2y=0. The sum Y satisfies
\prod_{i=1}^{6}(D^2+\omega_i^2)Y=0 (12th-order LODE for distinct \omega_i).
Refs: Wikipedia (Linear differential equation; Harmonic oscillator).
