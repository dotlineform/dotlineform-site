# Manifest CSV Integration

> Export meta Timestamp: 2025-08-29 15:37:21 BST What’s included: output/manifest.csv written during generation: filename,order,strategy,modes,auto_levels,source_files

## Why a Manifest?

To trace how each composite was constructed. It records the permutation order, strategy, modes used, auto-levels setting, and the exact source files used in that order.

## Fields

| Field | Description |
| --- | --- |
| `filename` | Saved PNG filename for the output. |
| `order` | Index order of sources (0-based), matching the deterministic (alphanumeric) load order from `data/input`. |
| `strategy` | `SEQUENTIAL` or `AVERAGE`. |
| `modes` | For `SEQUENTIAL`, per-layer modes used (includes `BASE` for the first layer). For `AVERAGE`, `N/A`. |
| `auto_levels` | `AutoLevels`, `AutoContrast`, or `None`. |
| `source_files` | Comma-separated original filenames in applied order. |

## Notes

- The sketch ensures deterministic load order by sorting filenames before reading.
- CSV fields are quoted if they contain commas/quotes to remain Excel-safe.

Ask if you want an additional column for per-layer *opacity* or *weights*, or a separate combinations-only generator.
