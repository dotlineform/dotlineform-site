# Coincidence Inevitability

**Claim:** In sufficiently dense streams of experience, *improbable* coincidences are *inevitable*. Their felt “synchronicity” arises when those coincidences pass through perceptual and narrative filters that confer *meaning*.

- **Mathematics:** Littlewood’s back‑of‑envelope law predicts “one‑in‑a‑million” events monthly, given everyday exposure to ~10<sup>6</sup> micro‑events.
- **Mind:** Predictive processing prioritises events that match active priors → salience → memory consolidation.
- **Information:** In complex systems, shared structure (mutual information) makes cross‑system matches common.

## 1) Littlewood’s Law — the Back‑of‑Envelope

Define a “miracle” as p ≈ 10<sup>−6</sup>. Assume you witness ~1 event/second while engaged with the world for ~8 hours/day.

- Events/hour ≈ 3×10<sup>4</sup>
- Events/day ≈ 2.4×10<sup>5</sup>
- Events/month (≈4 weeks) ≈ ~10<sup>6</sup>

Expectation: about one such “miracle” per month. The point is not precision but *scale*: long enough sequences make the unlikely routine.

## 2) Inevitability Meets Perception

The brain is a correlation‑first device. Active goals and concerns create *priors* that tune attention. When an external pattern fits those priors, prediction error drops and the event feels uncanny. Emotion tags the episode; reconsolidation stabilises the memory.

## 3) Complexity & Information

High‑dimensional environments contain vast combinatorial possibilities. Independent processes can still show *mutual information* when they share hidden structure (culture, language, seasonality). As dimensionality and volume rise, so does the chance of striking overlaps.

## 4) Practical Guidance

1. **Expect coincidences** as background texture; be surprised by *useful* ones, not their existence.
2. **Annotate salience**: what prior or goal did this match? (Name it.)
3. **Check base rates** using simple heuristics (weekly frequency × feature overlap).
4. **Act minimally**: take one reversible step; evaluate outcome in 24–72h.
5. **Maintain a null log**: record a few random “null windows” to calibrate how often weak coincidences occur.
