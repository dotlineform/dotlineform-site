# Repository Maintenance

This page preserves actionable maintenance gaps found during normal work. The owning durable documentation contains the explanation and remains authoritative.

## Follow-Ups

- Data Sharing — [row selection does not reach document apply](/docs/?scope=studio&doc=data-sharing-overview#known-gaps-and-pressure-points)
- Analytics — [offline fallback models are inconsistent](/docs/?scope=studio&doc=analytics-overview#known-weak-spots)

- GitHub Actions — [obsolete push-event condition remains](/docs/?scope=studio&doc=github-actions#weak-spots)

## Rules

- Link to the owning gap comment; do not repeat its explanation here.
- Include concrete follow-ups, not every general weak spot.
- Do not add task lists, implementation notes, priorities, or file inventories.
- When work becomes timely, add one bounded delivery and link it from the owning gap comment.
- When resolved, update or remove the owning gap comment and remove its link here.

Git preserves completed history. This page does not archive finished maintenance work.
