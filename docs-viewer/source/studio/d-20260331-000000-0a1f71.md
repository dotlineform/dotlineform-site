# Srcset Builder

## Purpose

`studio/services/media/make_srcset_images.py` converts staged source images into the configured primary and thumbnail variants.

The normal Work/detail path is the scoped catalogue build, which resolves source images, calls the derivative builder, copies public thumbnails, and reports readiness. Use the standalone command for explicit media maintenance or debugging, not as a second catalogue workflow.

## Command Shape

```bash
$HOME/miniconda3/bin/python3 studio/services/media/make_srcset_images.py INPUT_DIR OUTPUT_DIR JOBS
```

Omitting the directories uses the configured Work staging/output roots. `--dry-run` previews derivative writes and source cleanup.

Optional selected-ID and success-ID manifest paths plus the default worker count are read through names defined in `_data/pipeline.json`. Run `--help` and inspect `make_srcset_images.py` for the exact current argument/environment contract.

## Ownership Boundary

The builder owns image decoding, scaling/cropping, encoding, variant filenames, parallel execution, and staged-source cleanup. It does not:

- read or write canonical catalogue records;
- select publication scope;
- generate public JSON or search;
- copy final assets into R2;
- promote confirmed media versions.

Those responsibilities belong to catalogue build planning and [Publish Media To R2](/docs/?scope=studio&doc=d-20260508-093841-3e27b1).

Variant widths, suffixes, subdirectories, format, codec, quality, and compression come from the shared pipeline config. Changing them is a pipeline/public-runtime change, not a local flag tweak.
