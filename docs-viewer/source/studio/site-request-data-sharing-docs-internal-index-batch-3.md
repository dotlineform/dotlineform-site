### Batch 3: Data Sharing Consumer Migration

Summary: Move Data Sharing selectable-record, export, and returned-package review flows to the source metadata helper while preserving current behavior.

| ID | status | action |
| --- | --- | --- |

## Steer for these tasks

- Preserve current Data Sharing behavior and package formats.
- This is a dependency migration, not a UI or workflow redesign.
- Do not make Data Sharing consume public tree/search/recently-added payloads or manage/local generated indexes as substitutes.
- Keep scope handling config-driven and shared.

## Deliverables

- Migrated selectable-record reads.
- Migrated export/package preparation reads and source-text extraction.
- Migrated returned-package review context.
- Updated focused tests around migrated behavior.

## Implementation and policy guidance

- Keep Data Sharing source metadata calls behind the Batch 2 helper API.
- Delete old generated-artifact reads instead of wrapping them with aliases.
- Maintain current error/warning wording where practical unless the old wording explicitly names a generated artifact that is no longer the source.

## Proposed verification set

- `$HOME/miniconda3/bin/python3 -m py_compile` for changed service and test files.
- Focused pytest for docs export, docs import, Data Sharing package helpers, and API wrappers touched by the migration.
- `git diff --check`.
- No generated writes.
