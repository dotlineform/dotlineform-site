# Docs Viewer Route Dispatch Scripts

## `docs-viewer/services/docs_management_routes.py`

Purpose: single source of truth for management endpoint paths.

Ownership: owns endpoint path constants and the GET, POST, and OPTIONS allowlists.

Responsibilities:

- defines `/health`, `/capabilities`, generated read paths, source-config paths, import paths, source-editor paths, mutation paths, rebuild/audit paths, and scope lifecycle paths
- exposes `GET_PATHS`, `POST_PATHS`, and `OPTIONS_PATHS` for HTTP routing
- keeps compatibility aliases explicit

Not responsible for:

- endpoint behavior
- request validation beyond path allowlisting
- response payload shapes

## `docs-viewer/services/docs_management_service.py`

Purpose: POST route dispatcher for Docs Viewer management workflows.

Ownership: owns mapping POST endpoint paths to the focused service function that handles the workflow.

Responsibilities:

- dispatches source-editor rebuilds and source-open requests
- dispatches broken-links audit requests and attaches activity rows
- dispatches source-config settings writes and rebuild follow-through
- dispatches source import and import activity rows
- dispatches create, metadata, viewability, move, delete, and scope lifecycle actions
- invokes full same-scope rebuilds for `POST /docs/rebuild`

Not responsible for:

- HTTP socket handling
- generated read GET dispatch
- implementing source mutation planners
- implementing builder commands

## `docs-viewer/services/docs_management_context.py`

Purpose: shared context helpers for management service modules.

Ownership: owns repo/root detection, origin checks, path formatting, compact logging, and shared constants.

Responsibilities:

- finds and validates the repo root
- exposes local write-service constants such as log and watcher-suppression directories
- validates allowed origins for browser requests
- formats repo-relative paths for JSON responses
- writes compact management logs through `log_event`
- provides helper values such as the default Markdown app environment key

Not responsible for:

- endpoint-specific validation
- generated data parsing
- source mutation planning
