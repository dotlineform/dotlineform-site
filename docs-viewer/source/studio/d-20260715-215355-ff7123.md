# Local Tree Move Projection Concept

## Problem

A successful single-document drag/drop currently commits a bounded parent change on the server and then refetches and redraws the complete index in the browser. That resets unrelated interface context and makes a small structural change feel heavier than it is.

## Desired Result

- The server remains authoritative for validation, source writes, and rebuilds.
- The committed result updates the existing browser model and rendered tree.
- The moved subtree keeps its node identity and lands in deterministic order.
- The displayed document and unrelated expansion state do not change.
- A complete index reload is explicit error recovery, not the routine success path.

## Boundary

This feature improves the current one-document move workflow. It is not optimistic UI, a general incremental-mutation framework, or a multiple-selection feature.

Index Multiple Selection may later reuse the completed projection boundary for group movement, but that future consumer does not define this feature's value or delivery.
