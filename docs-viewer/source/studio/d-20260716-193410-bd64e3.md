# Copy Subtree To Scope Delivery

## Outcome

One singular management action copies a selected canonical document and all descendants into another existing writable scope as a complete independent subtree.

The original source is unchanged. The target receives new identities, target-root placement, remapped descendant parents and internal viewer links, target-default viewability, and one coordinated generated-docs/search rebuild. No overwrite, media copy, synchronization, or return path is introduced.

## Delivery Boundary

In scope:

- write-free preview and explicit confirmation;
- deterministic descendant expansion and copy ordering;
- new immutable identity allocation for every target document;
- canonical source transformation, hierarchy and internal-link remapping;
- public, local, and external-local target source roots;
- target-only source writes, watcher suppression, generated-docs/search rebuild, and activity reporting;
- one manage action, target-scope picker, result navigation, and focused durable documentation.

Out of scope:

- checkbox or arbitrary multi-root selection;
- copying within the same scope;
- overwrite, merge, update, synchronization, or return workflows;
- media, download, or interactive-asset byte copying;
- permanent copy provenance in canonical front matter;
- static HTML snapshots or Data Sharing packages.

## Code Authority To Inspect

- source loading, descendants, formatting, timestamps, identity allocation, and target-scope roots under `docs-viewer/services/`;
- the existing URL-aware viewer-link rewrite used by document-identity migration;
- management mutation planning, service dispatch, routes, coordinated write/rebuild, watcher suppression, and activity logging;
- management action definitions, capability projection, client calls, modal/controller ownership, and target navigation;
- focused source-model, mutation, route, service, and runtime-module tests.

These are responsibility areas, not a permanent module inventory.

### [ ] CPY-1 — Plan One Independent Subtree

- [ ] Resolve exactly one source root and expand its complete descendant set in deterministic source/tree order.
- [ ] Reject the current scope, unknown scopes, unavailable source roots, and targets that cannot accept managed canonical writes.
- [ ] Capture one copy timestamp and allocate a collision-free immutable id and matching target filename for every copied document without writing.
- [ ] Return a bounded preview naming the source scope/root, copied document count, target scope, and target-root placement.

**Checkpoint evidence:** the same unchanged source and target state produces one complete, write-free plan whose ids, paths, and hierarchy are internally valid.

### [ ] CPY-2 — Transform Canonical Sources

- [ ] Preserve each source body and front matter except `doc_id`, `added_date`, `last_updated`, `parent_id`, and `viewable`.
- [ ] Write the planned identity and copy timestamp, place the copied parent at target root, remap descendant parents, and apply the target scope's ordinary create viewability default.
- [ ] Rewrite only URL-aware Docs Viewer links whose source-scope targets are inside the copied set, preserving fragments and mapping them to the target scope and new ids.
- [ ] Leave links outside the set, semantic references, media tokens and paths, downloads, interactive assets, and source files unchanged; add no copy-provenance front matter.

**Checkpoint evidence:** every candidate target source parses cleanly, resolves its copied hierarchy, keeps external references intact, and contains no old copied-set parent or viewer-link target.

### [ ] CPY-3 — Apply One Target-Only Mutation

- [ ] Require explicit confirmation and revalidate the planned source set, target scope, identities, filenames, and absence of target collisions before the first write.
- [ ] Refuse every overwrite path and fail before mutation when the preview is stale or any planned target is no longer available.
- [ ] Write the complete planned source set through the canonical atomic-write owner and add watcher suppression for only the new target paths.
- [ ] Run one coordinated target generated-docs and search rebuild, record one bounded activity event, and return the new root id, created ids, target viewer URL, and rebuild result.

**Checkpoint evidence:** apply creates the complete target subtree once, changes no source-scope file, performs no overwrite, and leaves the target generated tree searchable and loadable.

## Done When

- one selected document and all descendants can be copied to any other currently writable configured scope;
- every copied document has a new immutable id, matching filename, copy-time `added_date`/`last_updated`, and target-default viewability;
- the copied parent is at target root and every descendant resolves only through new target ids;
- copied-set viewer links target the new subtree while external links, semantic references, and asset references retain their original meaning;
- preview writes nothing, apply overwrites nothing, repeated copies remain additive, and the source scope is unchanged;
- target source and generated documents are complete, searchable, and loadable after one coordinated rebuild;
- the management action uses the existing singular target, capability, modal, busy-state, and navigation owners;
- focused planner, mutation, route/service, generated-result, and runtime integration evidence passes;
- durable endpoint documentation describes the current copy contract and the feature documents are ready for explicit closeout review.
