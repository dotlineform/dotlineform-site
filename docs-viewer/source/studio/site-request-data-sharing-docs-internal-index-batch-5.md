## Steer for these tasks

- Durable docs should describe the final source-read contract, not the migration path.
- Generated Docs Viewer payloads should not be rebuilt for this request.
- Closeout should explicitly state that Data Sharing no longer depends on generated publication artifacts for docs metadata.

## Deliverables

- Updated durable Data Sharing and Docs Viewer docs.
- Cleanup scan results.
- Final verification log.
- Closed tracker and parent request.

## Implementation and policy guidance

- Keep generated-data docs clear that generated artifacts are publication/site-surface outputs.
- Keep Data Sharing docs clear that Data Sharing can share canonical-equivalent data externally and use returned data to update canonical docs source.
- Do not introduce compatibility language that suggests generated artifact reads remain acceptable Data Sharing metadata sources.

## Proposed verification set

- `git diff --check`.
- Focused stale-reference scans for generated publication artifact reads in active Data Sharing metadata paths.
- Focused docs scans for obsolete public-index or generated-index dependency language.
- Final focused pytest or smoke rerun only if Batch 5 changes executable code or if Batch 4 left a verification gap.
