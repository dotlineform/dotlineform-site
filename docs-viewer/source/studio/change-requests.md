# Change Requests

This section stores proposed and in-progress feature or infrastructure changes.

Use it for:

- product and workflow requests
- implementation planning requests
- UI request specs and task breakdowns
- request docs that should remain easy to find without being merged into stable reference docs yet

Do not use it for:

- stable runtime documentation
- stable design-system rules
- script reference docs
- changelog entries for already-completed work
