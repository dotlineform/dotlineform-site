## Steer for these tasks

- Tests should enforce the current architecture, not preserve historical generated-data shapes.
- Avoid brittle tests that assert implementation internals unrelated to the Data Sharing boundary.
- Negative-path tests should fail if generated publication artifacts become required for Data Sharing metadata again.

## Deliverables

- Positive behavior tests without generated publication artifacts.
- Negative dependency guards.
- Updated fixtures that distinguish source metadata behavior from Docs Viewer generated-data behavior.
- Batch 5 handoff with remaining docs and cleanup items.

## Implementation and policy guidance

- Keep permanent tests phrased around the Data Sharing source metadata contract.
- Do not keep temporary compatibility assertions.
- Generated publication artifacts may still appear in tests for Docs Viewer publication behavior; the guard applies to Data Sharing metadata workflows.
