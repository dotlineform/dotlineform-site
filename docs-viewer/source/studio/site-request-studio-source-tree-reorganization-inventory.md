# Studio Source Tree Reorganization Inventory

This inventory supports [Studio Source Tree Reorganization Request](/docs/?scope=studio&mode=manage&doc=site-request-studio-source-tree-reorganization) and [Studio Source Tree Reorganization Tasks](/docs/?scope=studio&mode=manage&doc=site-request-studio-source-tree-reorganization-tasks).

The classifications are path-family level decisions from the pre-move inventory, with current target paths recorded where moves have landed.
`STSR-004` resolved the ambiguous ownership rows before file moves began.
This inventory does not authorize compatibility paths, migration aliases, copied source artifacts, or serving old Studio-owned source from public asset paths.

## Resolved STSR-004 Decisions

- Current Jekyll-shaped Studio and UI Catalogue route hosts are Studio-owned cleanup targets, not public-site source. Active demo pages can remain until their move task, but `_layouts/studio.html` and Studio-specific includes have no final public owner.
- Docs Viewer shell, management shell, runtime config, UI text, CSS, reports, services, source scopes, and search builder logic are Docs Viewer-owned and live as a coherent internal subtree under `studio/docs-viewer/`. Public read-only routes may keep only minimal Jekyll adapters and generated runtime payloads outside `studio/`.
- Docs Viewer report source/config/runtime belongs with Docs Viewer. Report input data remains with its source domain, such as Studio-owned change request logs.
- `studio/workflows/change-requests/` source is Studio-owned workflow data. Generated log/report working output lives under Studio workflow generated paths unless a later task identifies a public generated payload required by a read-only route.
- `studio/data/canonical/catalogue-markdown/` and canonical catalogue JSON are Studio canonical data. Derived catalogue lookup payloads are Studio editor read models with a schema contract; disposable caches belong under `var/`, not tracked source.
- Checks, tests, fixtures, smoke helpers, projection contracts, public-surface audits, and public-site build/preview helpers are Studio/Codex development infrastructure, even when they validate published routes.
- Search ownership splits by domain: Catalogue search generation/config/policy belongs to Studio; Docs Viewer search generation/config belongs to Docs Viewer; public search route JavaScript and generated search payloads remain public runtime/output.
- `assets/docs/interactive/` is Docs Viewer interactive source asset input. It moves with Docs Viewer source/runtime unless a specific generated public copy is required for read-only published docs.
- Public route directories such as `analysis/`, `library/`, `docs/`, `search/`, `catalogue/`, `works/`, `series/`, `data/`, and `logs/` remain public route surfaces only when they publish pages or generated runtime payloads. Studio-only local route shells do not remain there as final source.
