## Steer for these tasks

- This batch creates the source-read capability only; it does not migrate Data Sharing consumers yet except where tests need direct helper coverage.
- Do not add `metadata-index.json`, `tooling-index.json`, or any other generated metadata output.
- Do not update scope lifecycle generated-output manifests because this request adds no generated data artifact.
- Keep helper APIs narrow and shaped around current Data Sharing workflows.

## Deliverables

- Docs Viewer-owned source metadata helper API.
- Source rendering/source-text helper for export use.
- Focused helper tests and negative generated-artifact dependency tests.
- Batch 3 handoff listing exact consumer call sites to migrate.

## Implementation and policy guidance

- Use structured source parsing and shared Docs Viewer rendering helpers.
- Keep filesystem path knowledge inside Docs Viewer-owned helpers.
- Generated publication artifacts are not valid inputs for this helper.
- Do not preserve old read paths as fallback behavior.
