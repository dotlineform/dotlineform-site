### general steer

- Data Sharing is a first-class canonical-data service, not a generated-data workflow.
- In this request, generated data means artifacts produced for publication, preview, management, report, route, search, or other site-surface workflows, regardless of whether they are public, private, manage-only, local-only, or checked in.
- Data Sharing must use canonical-equivalent docs data from scope config, docs source front matter, and docs source body content processed through shared Docs Viewer parsing/rendering code.
- Preserve current Data Sharing behavior, UI options, package workflows, package formats, and selectable-record fields, including `published`.
- Cover every configured Docs Viewer scope through shared scope-config-driven code. Do not create scope-specific modules or repeated `library`/`analysis` code blocks.
- Do not add `metadata-index.json`, `tooling-index.json`, or another generated metadata JSON as the Data Sharing source.
- Do not add compatibility shims or dual-read fallbacks to generated publication artifacts.
- Source docs should be updated; generated Docs Viewer payloads should not be rebuilt for this request unless an unrelated local watcher does so.
