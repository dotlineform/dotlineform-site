# Docs Viewer Audit Scripts

## `docs-viewer/services/docs_management_broken_links_service.py`

Purpose: route helper for the broken-links management endpoint.

Ownership: owns the service-facing adapter for `POST /docs/broken-links`.

Responsibilities:

- validates the request body shape needed by the audit endpoint
- invokes the shared docs broken-links audit for the selected scope
- returns endpoint-ready JSON payloads
- leaves activity attachment to the route dispatcher

Not responsible for:

- implementing link scanning
- writing source docs
- rebuilding generated output

## `docs-viewer/services/docs_broken_links.py`

Purpose: shared docs broken-links audit engine and terminal entrypoint.

Ownership: owns source-doc link scanning and missing-target issue detection.

Responsibilities:

- scans docs source for supported docs links and reference targets
- reports missing target issues
- supports terminal-backed audit use
- provides the audit behavior reused by the management endpoint

Not responsible for:

- HTTP routing
- Studio Activity row construction
- mutating source or generated data

## `docs-viewer/services/docs_activity.py`

Purpose: Docs Management Studio Activity helper.

Ownership: owns activity row construction for covered docs management actions.

Responsibilities:

- attaches import-source activity rows after successful create/overwrite imports
- attaches broken-links audit activity rows with checked and broken-link counts
- suppresses activity rows for previews, confirmation-only responses, and invalid context
- records source refs, warning status, and record groups for docs actions

Not responsible for:

- executing import or audit workflows
- serving activity UI
- creating generic non-docs activity records
