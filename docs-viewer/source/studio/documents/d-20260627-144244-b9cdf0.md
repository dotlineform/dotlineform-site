# Selectable List

Use the selectable list when a route needs single or multiple selection from normalized records. It can also own generic parent/child ordering, collapse state, descendant selection, and indeterminate parent state.

## Authority

- behaviour: `shared/frontend/js/selectable-list.js`
- styles: `shared/frontend/css/selectable-list.css`
- current adapters: `analytics-app/app/frontend/js/data-sharing-prepare-render.js` and `data-sharing-review-render.js`

Search imports from `/shared/frontend/js/selectable-list.js` for the current consumer set.

## Stable Structure

Routes provide items with a stable id and visible label. Optional accessors or item fields provide metadata, search text, parent id, and disabled state. The code owns the exact option and controller API.

The component owns:

- row and checkbox rendering
- single or multiple selected-id state
- select-all and clear behaviour
- optional tree ordering, collapse, cascade selection, and indeterminate state
- callbacks containing selected ids and selected items
- update and destroy lifecycle

The route owns:

- APIs, loading, errors, adapter/profile/scope state, and filtering policy
- mapping domain records into the component item shape
- storing selection in route state
- deciding which workflow actions selection enables
- prepare, review, apply, or other mutations

Network loading state stays outside the component. Empty, filtered-empty, and list-construction states may be rendered inside it.

## Extension Boundary

The component returns selected ids as canonical UI state and treats attached domain data as opaque. It should not inspect Docs Viewer hierarchy, Data Sharing adapters, staged files, or tag semantics.

Generic accessors and tree behaviour belong here. Domain columns, row actions, editable fields, and workflow-specific selection rules generally belong in an adapter or a different component.

## Method And Weak Spots

- Tree behaviour is derived from the current item set; incomplete or cyclic parent data is an adapter/data problem.
- Select all operates on the enabled visible item set defined by the component and its generic hooks. Routes must make any domain exclusions explicit before rendering.
- Selection is id-based and held in browser memory. The route remains responsible for reconciling refreshed data.
- This is a selectable list, not a table or record-action system. Use [Record List And Actions](/docs/?scope=studio&doc=d-20260614-201632-79c9cb) when fixed columns and commands are the primary need.

## Verification

Protect:

- empty and construction states
- disabled rows
- single and multiple selection
- select all and clear
- tree order, collapse, cascade, and indeterminate state
- callback payloads
- refreshed item/selection reconciliation
- listener cleanup

Test route normalization and workflow payload construction separately from component interaction.
