# Extract Imported SVG To Published Media

## Outcome

Docs Import stores sanitized SVG once at the target scope's configured published `img` location and writes a logical media token into canonical Markdown instead of retaining a large inline `<svg>` block.

This applies both to a standalone staged SVG and to inline SVG encountered while converting imported HTML. It extends the existing inline-raster extraction model rather than creating a separate diagram store.

## Import Contract

- SVG is parsed and sanitized before publication. Script blocks, event-handler attributes, and other rejected content never reach the published artifact or canonical Markdown.
- Each extracted SVG receives a deterministic, collision-safe media identity associated with the target document. Overwrite imports retarget generated identities to the existing canonical `doc_id` in the same way as current extracted raster media.
- The target scope's `img` configuration selects repository, external-local, or R2 publication. Import logic does not choose a provider.
- Media publication and byte verification complete before source is written.
- Canonical Markdown contains an ordinary image reference such as:

<pre><code>![Diagram](&#91;&#91;media:docs/&lt;scope&gt;/img/&lt;identity&gt;.svg&#93;&#93;)</code></pre>

- A useful SVG `<title>` supplies the initial image alt text when available. The author can refine that text after import.
- Existing manually authored inline SVG remains valid; this feature changes import conversion, not the general Markdown authoring contract.

## Boundaries

In scope:

- standalone SVG source import;
- inline SVG extracted from imported HTML;
- sanitized SVG serialization, published-media planning, token rewriting, collision handling, and provider-independent apply;
- focused import preview, apply, failure, and rendered-result verification.

Out of scope:

- scanning or migrating existing source documents that already contain inline SVG;
- extracting SVG from arbitrary existing Markdown;
- a Mermaid renderer or registered Mermaid build producer;
- source-editor **Add image** or **Add file** actions;
- a general SVG editor, optimizer, formatter, or asset registry.
