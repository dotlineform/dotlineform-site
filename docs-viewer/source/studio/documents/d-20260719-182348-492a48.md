# HTML Import Mapping

Docs Import converts parsed HTML through one closed, code-owned mapping in `docs-viewer/services/docs_html_markdown.py`. The mapping is shared by staged HTML and format adapters such as Word import; it produces canonical Markdown and warnings but never writes source or publishes media.

This page owns the intended conversion and degradation rules. The converter and its focused tests remain the exact authority.

## Title And Block Mapping

- Import title preference is non-empty `<title>`, then the first non-empty `<h1>`, then the caller's filename-derived fallback.
- `<h1>` through `<h6>` retain their Markdown heading levels. `<p>`, `<section>`, `<article>`, `<body>`, `<div>`, and `<footer>` retain readable block order.
- `<blockquote>` becomes Markdown quoted lines; `<pre>` becomes a fenced code block.
- `<ul>` and `<ol>` become Markdown lists, including nested lists. A `div.legend` wrapper is flattened to comma-separated text.
- `<details>` becomes a level-3 heading from `<summary>` followed by its body.
- A `div.key` becomes a blockquote callout.
- Unknown containers recurse into their readable children rather than introducing new Markdown syntax.

## Inline Mapping

| HTML | Markdown behavior |
| --- | --- |
| `<strong>`, `<b>` | bold |
| `<em>`, `<i>` | emphasis |
| `<code>` | inline code |
| `<a href>` | Markdown link; link text falls back to the URL |
| `<br>` | line break |
| `<sub>`, `<sup>` | retained as safe inline HTML |
| plain absolute URLs | autolinked outside links and code |
| `<img>` | external URLs and inline data URLs remain image candidates for later media planning; other sources degrade to their source or alt text |
| inline `<svg>` | retained as contiguous HTML for the existing sanitizer and media planner |

Image extraction, sanitization, target naming, and publication belong to [Media Handling](/docs/?scope=studio&doc=d-20260514-184303-7914e2), not this mapping.

## Tables, Prompt Metadata, And Sanitization

- A simple table uses its first row as the Markdown header and pads shorter rows to the widest row. Multiple block children within a cell are separated with inline `<br>`; nested list items are flattened to semicolon-separated text because Markdown tables do not support block lists.
- A table containing `rowspan` or `colspan` becomes plain text with a conversion warning.
- Prompt/meta blocks are recognized through `prompt`, `meta`, or `shade` class tokens, related prompt/meta class names, or the text prefixes `[prompt]`, `Original prompt`, and `Follow-up`. They are omitted by default and become blockquotes only when the operator includes prompt metadata.
- HTML comments and `head`, `script`, `meta`, and `title` content do not enter Markdown. Non-SVG `<style>` content is also dropped.
- Serialized non-SVG HTML drops event attributes plus `style`, `class`, `id`, `target`, and `rel`. Inline SVG retains its required structural attributes but drops event attributes and scripts.

## Extension Rule

This is a semantic content mapping, not a general HTML-preservation or layout-fidelity pipeline. Add a rule only for a representative content structure with an explicit canonical-Markdown outcome, safe degradation, and focused tests. Format adapters should supply semantic HTML to this converter unchanged instead of adding a second renderer or a chain of output-specific repairs.

[Docs Import Architecture](/docs/?scope=studio&doc=d-20260514-164722-3de65e) owns the adapter, preview, planning, and mutation boundaries around this converter.
