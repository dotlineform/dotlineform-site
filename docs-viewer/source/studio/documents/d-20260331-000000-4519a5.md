# Tag Registry

## Capability

`/analytics/tag-registry/` maintains canonical tags: browse/filter, import, create, edit, delete, and demote a tag into an alias.

Canonical identity has wider consequences than visible registry rows. Rename/delete/demotion can affect aliases and assignments, so the server planners own impact calculation and cross-file mutation.

## Workflow

```text
load registry + aliases + assignments
  -> filter/search or open mutation modal
  -> validate candidate ID/group/description
  -> preview destructive or cross-file impact
  -> confirm
  -> server planner + atomic apply
  -> reload canonical data + activity evidence
```

When the Local Analytics service is unavailable, supported simple operations produce manual JSON patch guidance. That is a fallback handoff, not a canonical write or guaranteed multi-file transaction.

## Durable Rules

- canonical IDs are `<group>:<slug>` and group must match the ID prefix;
- imports support the modes implemented by the registry workflow; server validation remains authoritative;
- deletion and demotion must account for alias/assignment references;
- demotion replaces a canonical tag with an alias targeting selected remaining canonical tags;
- local-server mutations record one initiating action context in Admin activity.

Exact limits, import shapes, and effect payloads live in `tag_source_model.py`, focused planners, and tests.

## Code Map

- template/controller: `tag-registry.html`, `tag-registry.js`
- state/render/modal/import/save modules: `analytics-app/app/frontend/js/tag-registry-*`
- HTTP boundary: `tag_write_api/registry.py`, `tag_write_api/promotions.py`
- planners: `tag_registry_mutations.py`, `tag_promotion_mutations.py`
- canonical source: `analytics-app/data/canonical/tag-registry.json`

The route root follows the shared Analytics ready/busy contract. Import/edit modal state is route mode; service availability reflects direct-write capability.

## Weak Spots And Change Method

- Manual patch fallback cannot safely reproduce every cross-file server mutation; keep its supported surface explicit.
- The controller remains a large coordinator even after state/render/workflow modules were split.
- Visible copy is code-owned in `analytics-ui-text.js`, so it can drift from server terminology.

Change the domain planner and tests before the modal/controller. Add UI detail here only when it reveals a durable workflow or ownership boundary.
