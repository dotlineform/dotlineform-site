## Status

- The feature is ready and not active. Its complete implementation boundary is the linked delivery.
- Staged `.mmd` intake uses **Add image…**, backed by the repository-pinned Mermaid CLI and one registered producer for interactive, targeted-document, watcher, and full Docs builds.
- The source/publication refactor established the convention: canonical `.mmd` belongs in configured source `media/mermaid/`, and derived SVG is written directly to that scope's configured published `svg` location.
- The full-build producer seam is present but remains a no-op until a Mermaid producer is registered; targeted document builds currently skip media producers and must render referenced Mermaid outputs before their document payload.
- The first proof diagrams belong to **Configuration And Extension Points** and **Docs Import**. Rendered Document Info links their verified Mermaid sources to VS Code; source-mode info remains semantic-token search.
- No package manifest, installed dependency, registered producer, source folder, proof diagram, watcher integration, or automated build integration has been implemented yet.
- D2 is a possible alternative if a concrete diagram exposes a Mermaid layout or expression limit.
