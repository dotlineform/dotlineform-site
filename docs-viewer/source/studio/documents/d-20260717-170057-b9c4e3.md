## Frozen `docs_scopes_v2` Shape

SSR-2 replaces `source`, `output`, `search_output`, `publish_output`, `publish_search_output`, `media_path_prefix`, `import_media_storage`, and `external_data_root` with role-first objects. Presentation and policy fields such as `scope_id`, `scope_type`, route settings, document defaults, viewability lists, and status settings remain.

```json
{
  "schema_version": "docs_scopes_v2",
  "scopes": [
    {
      "scope_id": "studio",
      "scope_type": "local",
      "source": {
        "location": {
          "provider": "repository",
          "path": "docs-viewer/source/studio"
        },
        "documents_path": "documents",
        "build_media": {},
        "sub_scopes_path": "sub-scopes"
      },
      "published": {
        "documents": {
          "location": {
            "provider": "repository",
            "path": "docs-viewer/published/docs/studio"
          }
        },
        "search": {
          "location": {
            "provider": "repository",
            "path": "docs-viewer/published/search/studio/index.json"
          }
        },
        "media": {
          "img": {
            "reference_prefix": "docs/studio/img",
            "location": {
              "provider": "repository",
              "path": "docs-viewer/published/docs/studio/media/img"
            },
            "served_path_prefix": "/docs/media/studio/img",
            "build_inputs": {}
          }
        }
      },
      "public_projection": null,
      "sub_scopes": []
    }
  ]
}
```

Schema rules:

- `location.provider` is exactly `repository`, `external_local`, or `r2`; `location.path` is a confined filesystem path or object-key prefix appropriate to that provider.
- `source.location.path` is the owned scope container. `documents_path`, `build_media[*].path`, and `sub_scopes_path` are confined relative paths beneath it.
- Each `build_media` entry names `path`, `producer`, and `publishes_to`; no entry is added until a real build-source type exists. A later Mermaid entry can publish to the one `img` target without moving ready images.
- `published.documents` and `published.search` are the Docs Viewer local consumer-ready payload locations. Public copies do not overload them.
- Every configured published media type has one `reference_prefix`, one `location`, one `served_path_prefix`, and zero or more explicit `build_inputs`. There is no scope-wide media storage mode.
- `public_projection` is either `null` or contains independent `documents` and `search` repository locations. It does not select the media provider.
- A sub-scope repeats the same `source`, `published`, and optional `public_projection` objects beneath its own configured roots.
- External-local policy requires every source and media location to use `external_local` and remain beneath the configured external scope folder; its document/search payload workspace remains beneath the external Docs Viewer root.
- Browser-safe projection exposes viewer settings, logical URLs, and served paths only. It never exposes source paths, filesystem locations, credentials, or write targets.
