# Standalone Docs Viewer

## Purpose

Package and release a self-contained local application as the first independent consumer of the explicit Docs Viewer core. The [Docs Viewer Core Refactor](/docs/?scope=studio&doc=d-20260718-114645-7b9d24) owns extracting and proving that boundary for both compositions.

## Feature Documents

- [Concept](/docs/?scope=studio&doc=d-20260718-105957-c72b2a) — the core problem, target composition, current assessment, standalone proof, full-application benefit, and guardrails.
- [Docs Viewer Core Refactor](/docs/?scope=studio&doc=d-20260718-114645-7b9d24) — prerequisite feature for core extraction, integrated recomposition, main-owned planning, and working notes.
- [Docs Viewer Roadmap](/docs/?scope=studio&doc=d-20260714-234030-1e0af6) — scheduling, dependencies, and suggested feature order.

## Feature Boundary

The standalone application retains full management of external-local scopes, including direct local import and export. It has no repository-backed scopes, public projections, Data Sharing, or Docs Review context.

The public dotlineform repository remains the source, collaboration, and release platform. An installed release bundles its own runtime and dependencies and resolves no files from a repository checkout.

## Status

The feature is proposed. Its prerequisite Core Refactor is also proposed, so standalone packaging and release work is not ready to begin. No standalone architecture or delivery document exists yet.
