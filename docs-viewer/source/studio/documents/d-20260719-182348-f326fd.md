# Word Import Style Mapping

The Docs Import Word adapter uses one code-owned Mammoth style map. It converts Word paragraph styles into a small semantic HTML vocabulary, then passes Mammoth's HTML unchanged to the existing [HTML Import Mapping](/docs/?scope=studio&doc=d-20260719-182348-492a48).

This page owns the deliberate Word style decisions. `docs-viewer/services/docs_import_docx.py` and its focused tests remain the exact authority.

## Current Mapping

| Word paragraph style | semantic HTML | resulting Markdown role |
| --- | --- | --- |
| `Title` | `<h1 class="dlf-docx-title">` | document H1 and import title marker |
| `Subtitle` | `<p class="dlf-docx-subtitle">` | ordinary prose below the title |
| `Heading 1` | `<h2>` | section heading at level 2 |
| `Heading 2` | `<h3>` | section heading at level 3 |
| `Heading 3` | `<h4>` | section heading at level 4 |
| `Heading 4` | `<h5>` | section heading at level 5 |
| `Heading 5` | `<h6>` | section heading at level 6 |
| `Heading 6` | `<h6 class="dlf-docx-heading-clamped">` | level-6 heading with a lossy-clamp warning |

The map registers both Word style IDs and style names for these built-in styles. Mammoth's default style map remains enabled for ordinary semantics such as paragraphs, emphasis, lists, links, and tables.

## Title And Degradation Rules

- A mapped Word `Title` supplies the import title. The adapter may inspect its semantic class but does not remove or rewrite the resulting H1.
- Without a mapped `Title`, the humanized staged filename supplies the import title. The first Word heading remains a section heading; it is not promoted into a document title.
- Word has six heading styles but Markdown has only six heading levels. Because `Title` occupies level 1, `Heading 6` is retained at level 6 and reported as a lossy clamp.
- An unmapped paragraph style follows Mammoth's default degradation and its conversion message remains visible as an import warning.

## Style-Map Boundary

- The repository map is the only accepted style map. Document-embedded and user-supplied maps are disabled.
- Style meaning is decided before HTML conversion. The adapter does not repair or post-process Mammoth's generated HTML.
- External file access is disabled. Embedded-media handling remains a separate import-media concern rather than a style rule.
- Adding or changing a mapping requires a representative Word structure, one deliberate semantic decision, and focused adapter plus Markdown evidence. Do not accumulate document-specific fixes.

[Word Document Import](/docs/?scope=studio&doc=d-20260718-233015-a1836e) owns the active feature boundary, and [Word Document Import Delivery](/docs/?scope=studio&doc=d-20260719-174421-aa4ff0) owns its implementation checkpoints.
