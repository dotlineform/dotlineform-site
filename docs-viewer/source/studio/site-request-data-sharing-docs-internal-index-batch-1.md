## Steer for these tasks

- This batch is source review and contract locking only.
- Do not implement compatibility reads or temporary fallbacks.
- Do not add generated metadata outputs.
- Preserve current Data Sharing behavior, including `published`, UI options, package formats, selection behavior, and returned-package review workflows.
- Treat generated publication artifacts as out of bounds even when they are local, manage-only, private, or checked in.

## Deliverables

- Audit notes in this batch document.

- Final helper API/owner decision.
- List of implementation touch points and tests for Batch 2 through Batch 4.

- Keep Data Sharing as the consumer of a service helper, not a filesystem/path reconstruction layer.
