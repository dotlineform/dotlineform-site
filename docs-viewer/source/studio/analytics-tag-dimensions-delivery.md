# Small Tag Dimension Set Delivery

## Outcome

Define a very small dimension set for one concrete analytical question, score a bounded set of catalogue identities through native controls, and use the result in one comparison or review that demonstrates whether the data is useful.

This is a candidate first delivery, not active work.

## In Scope

- one clearly stated analytical question;
- a deliberately small set of dimensions;
- explicit definition ownership and subject eligibility;
- one native representation and validation rule per dimension;
- distinct applicability, missing, and scored states;
- one bounded scoring and review workflow;
- one named projection or comparison with declared information loss;
- evidence that the result does or does not support a useful decision.

## Not In Scope

- a complete dimension ontology;
- every representation family;
- cross-domain scoring;
- generalized visualization infrastructure;
- automatic LLM scoring;
- promotion tiers that have not been justified by use;
- forced numeric normalization for categorical or multi-part meanings.

## Completion Decision

Close the delivery by deciding whether to retain, revise, or discard the model. Generalize registries, controls, projections, or assistance only from the structure the proving workflow actually needs.

## Durable Owners

If retained, shipped definitions, score schemas, and extension rules move into current Analytics registry and workflow documentation. This delivery keeps only unresolved follow-up direction.
