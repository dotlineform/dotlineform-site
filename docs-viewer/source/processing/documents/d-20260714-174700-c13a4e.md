# curve boundary function



parameter values control the boundaries of the ink line

quick - short narrow 
natural - contained
wild - could go anywhere 

so define circle segments for each of these 
return a curve options object

Boundary function just takes a single parameter ‘start point’

then we can draw the curve


 <span style="background-color: #00C6BD;">
     create a new function demo_createLineBounded which draws a vector field line (like demo_CreateLine) using the setLineBoundary function

 </span>
Below is an example of a new function, **demo_createLineBounded()**, that uses the existing **setLineBoundary** function to create a bounded curve (a CurveOptions object) and then draws a vector field line using **createLine_VectorField**.

**Explanation**
	•	**LineOptions:**
Specifies the number of sample points and the base line width.
	•	**Ink/VectorInkOptions:**
Combines generic ink parameters (like inkFraction, decayType, exponent, etc.) with vector-specific parameters (distortionScale and noiseScale). These control how the stroke decays and how much distortion is applied.
	•	**setLineBoundary:**
This function (already defined elsewhere) creates a bounded curve (CurveOptions object) based on a given start point. It automatically calculates default control points and an end point relative to that start.
	•	**BezierCurve:**
The **createBezierCurve** function (also defined elsewhere) converts the CurveOptions into a BezierCurve object that is used by the drawing function.
	•	**createLine_VectorField:**
Draws a smooth stroke along the Bezier curve and then applies a noise-based distortion to simulate organic, wavy ink edges.
	•	**Labeling:**
A text label (“Vector Field Bounded”) is drawn to the left of the start point for clarity.

Call **demo_createLineBounded()** (for example, from your setup or key press handler) to see the bounded vector field line in your sketch.

—————

	•	**Function Signature Change:**
The function now accepts a **String style** in addition to the start point.
	•	**Conditional Branching:**
	•	If **style** is "quick", it uses smaller offsets for a rapid, short curve.
	•	If **style** is "natural", it uses moderate offsets for a balanced curve.
	•	If **style** is "wild", it uses larger offsets for a more exaggerated, dynamic curve.
	•	If an unrecognized style is provided, the function defaults to the "natural" style.

This modification lets you control the curvature style by passing the desired style value when calling **setLineBoundary**.

 <span style="background-color: #00C6BD;">
     the code produces the same drawing each run. why?
 </span>

The drawing is identical each run because the parameters determining the curve and its distortion are fixed. For example:
	•	**setLineBoundary:** When you call it with a specific style like “natural,” it always generates the same control points and endpoint relative to the start point.
	•	**Static Noise:** In your vector field function, you’re using a constant value (staticTime = 1.0) and not modifying the noise seed. The Processing noise() function is deterministic (producing the same output each run unless seeded differently).
 <span style="background-color: #FF9502;">
     	•	
 </span> <span style="background-color: #FF9502;">
     **No Random Seeding:**
 </span> <span style="background-color: #FF9502;">
      If you haven’t called randomSeed() or noiseSeed() with a dynamic value (like based on the current time), the pseudo-random number generators (both for random() and noise()) use their default seeds, which results in the same random sequence on each run.
 </span>

If you want a different drawing each run, you could:
	•	Call randomSeed(millis()) or noiseSeed(millis()) in your setup() function.
	•	Or incorporate some additional randomness in functions like setLineBoundary (e.g., by adding random offsets to the control points).

These changes would introduce variation so that the drawing isn’t identical every time.
