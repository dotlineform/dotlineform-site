# Processing in the Cloud — Batch Composites & Python Workflows

This guide shows two production‑ready routes: (A) Dockerized *Processing* (Java mode) for headless, repeatable batch composites; (B) Python/Colab workflows for image blending and stitching to video. It includes runnable snippets, job structure, and references.

## Executive Summary

### A. Dockerized Processing (Batch)

- Use processing-java to run/export sketches headlessly inside a container.
- Wrap with xvfb-run for virtual display when needed.
- Draw to `PGraphics` off‑screen buffers and save PNGs without a visible window.
- Emit a `manifest.csv` per frame/composite to log parameters.

### B. Python‑First (Colab/Jupyter)

- Blend images with Pillow/NumPy (or processing.py).
- Write frames; stitch with FFmpeg from Python.
- Parameterize runs (stride, weights, modes) and save manifests.
- Store outputs to cloud buckets (GCS/S3) for downstream tools.

## A. Batch Composites with Dockerized Processing

### A.1 Why use `processing-java`?

The official command‑line tool lets you *build, run, present, and export* sketches without opening the IDE. It supports passing args to your sketch and works on macOS, Windows, and Linux. See the Processing wiki for installation and options. [Sources: [Processing Wiki — Command Line](https://github.com/processing/processing/wiki/Command-Line)]

### A.2 Headless/Server environments

Many servers have no attached display. When your sketch uses AWT/OpenGL, run it inside a *virtual X display* via `xvfb-run`, or spin up a manual `Xvfb` display and export `DISPLAY`. [Sources: [Running without a Display](https://github.com/processing/processing/wiki/Running-without-a-Display)]

Tip:

For image‑only pipelines, prefer drawing to off‑screen

PGraphics

buffers using

createGraphics()

+

beginDraw()/endDraw()

, then

save()

the buffer. This avoids GUI windows entirely.

### A.3 Minimal Dockerfile

Base on an OpenJDK image (or Eclipse Temurin) and install `xvfb` + Processing CLI. Parameterize the Processing version at build time.

```
# Dockerfile
# Option 1: Eclipse Temurin (OpenJDK 17 LTS)
FROM eclipse-temurin:17-jdk

# Install xvfb & dependencies
RUN apt-get update -y  && DEBIAN_FRONTEND=noninteractive apt-get install -y --no-install-recommends     xvfb libxrender1 libxtst6 libxi6 fontconfig curl ca-certificates unzip  && rm -rf /var/lib/apt/lists/*

# Install Processing (processing-java) — set version via build arg
ARG PROC_VER=4.4.6
RUN curl -L -o /tmp/processing.tgz https://download.processing.org/processing-${PROC_VER}-linux-x64.tgz  && mkdir -p /opt/processing  && tar -xzf /tmp/processing.tgz -C /opt/processing --strip-components=1  && ln -s /opt/processing/processing-java /usr/local/bin/processing-java  && rm /tmp/processing.tgz

# App layout
WORKDIR /app
# Place your sketch folder at /app/SketchName/SketchName.pde (folder name == main .pde name)
COPY sketch /app/sketch

# Default command: run headless with xvfb
CMD xvfb-run --auto-servernum --server-args='-screen 0 1920x1080x24'     processing-java --sketch=/app/sketch --output=/app/build --force --run
```

Base image: [Eclipse Temurin (OpenJDK) official image](https://hub.docker.com/_/eclipse-temurin).

### A.4 Sketch template (off‑screen composites)

```
// File: sketch/BatchComposite/BatchComposite.pde
// Headless-friendly: draws to PGraphics and saves PNGs.
import java.util.Arrays;

String inputDir = System.getProperty("INPUT_DIR", "input");   // pass via -DINPUT_DIR=...
String outputDir = System.getProperty("OUTPUT_DIR", "output"); // pass via -DOUTPUT_DIR=...
int W = int(System.getProperty("W", "3840"));  // UHD by default
int H = int(System.getProperty("H", "2160"));
String blend = System.getProperty("BLEND", "AVERAGE");        // or SEQUENTIAL
float opacity = float(System.getProperty("OPACITY", "0.5"));  // used by SEQUENTIAL

void settings(){ size(16,16); } // tiny; we won't use the window

void setup(){
  // Create off-screen buffer
  PGraphics pg = createGraphics(W, H);
  pg.beginDraw();
  pg.background(0);

  // Load images
  java.io.File[] files = listFiles(inputDir);
  java.util.Arrays.sort(files);
  PImage[] imgs = new PImage[min(files.length, 16)]; // example: cap to 16 layers
  for (int i = 0; i < imgs.length; i++) imgs[i] = loadImage(files[i].getAbsolutePath());

  if(blend.equals("AVERAGE")){
    pg.blendMode(BLEND);
    // Average by accumulating with equal alpha
    float a = 255.0/imgs.length;
    for(PImage im : imgs){ pg.tint(255, a); pg.image(im, 0, 0, W, H); }
    pg.noTint();
  } else { // SEQUENTIAL with constant opacity
    for(PImage im : imgs){ 
      pg.tint(255, 255*opacity); 
      pg.image(im, 0, 0, W, H); 
    }
    pg.noTint();
  }

  pg.endDraw();
  java.io.File outDir = new java.io.File(outputDir);
  outDir.mkdirs();
  String outPath = outputDir + "/composite.png";
  pg.save(outPath);
  println("Saved: " + outPath);
  exit();
}

java.io.File[] listFiles(String dir){
  java.io.File d = new java.io.File(dir);
  if(!d.exists()) d.mkdirs();
  return d.listFiles((f)-> f.getName().toLowerCase().matches(".*\.(png|jpg|jpeg)$"));
}
```

### A.5 Running with parameters

```
# Build image
docker build -t processing-batch --build-arg PROC_VER=4.4.6 .

# Run (mount inputs/outputs and pass Java properties)
docker run --rm -v "$PWD/input":/data/in -v "$PWD/out":/data/out   -e "JAVA_TOOL_OPTIONS=-DINPUT_DIR=/data/in -DOUTPUT_DIR=/data/out -DW=3840 -DH=2160 -DBLEND=AVERAGE -DOPACITY=0.5"   processing-batch
```

### A.6 Stitch frames to video (FFmpeg)

Use the *image2* demuxer to turn numbered PNGs into a video. Specify input rate with `-framerate`; set output codec/rate as needed. [Sources: [FFmpeg Formats — image2](https://ffmpeg.org/ffmpeg-formats.html#image2-1), [FFmpeg docs](https://ffmpeg.org/ffmpeg.html)]

```
# Example: 25 fps UHD H.264
ffmpeg -framerate 25 -pattern_type glob -i 'out/frame_*.png'   -c:v libx264 -pix_fmt yuv420p -crf 18 -r 25 out.mp4
```

### B.2 Pure‑Python image blending → frames → FFmpeg

```
# %% install (Colab may already have pillow & ffmpeg)
# !pip -q install pillow numpy pandas

import os, glob, csv, subprocess
from PIL import Image
import numpy as np

INPUT_DIR = "input"
OUTPUT_DIR = "frames"
os.makedirs(OUTPUT_DIR, exist_ok=True)

# Example: weights for an average (equal weights also fine)
def normalize(w):
    w = np.array(w, dtype=np.float32)
    return w / (w.sum() if w.sum() else 1.0)

def composite(images, weights):
    arr = None
    w = normalize(weights)
    for i, path in enumerate(images):
        im = Image.open(path).convert("RGBA")
        a = np.array(im, dtype=np.float32) / 255.0
        rgb = a[..., :3] * a[..., 3:4]  # pre-multiply alpha
        if arr is None:
            arr = rgb * w[i]
        else:
            arr += rgb * w[i]
    out = np.clip(arr, 0, 1)
    out = (out * 255).astype(np.uint8)
    return Image.fromarray(out, mode="RGB")

# Batch: every N images => one composite frame
paths = sorted(glob.glob(os.path.join(INPUT_DIR, "*.*")))
N = 5  # group size
weights = [1.0]*N  # average
manifest_path = os.path.join(OUTPUT_DIR, "manifest.csv")
with open(manifest_path, "w", newline="") as f:
    writer = csv.writer(f)
    writer.writerow(["frame", "source_files", "weights"])
    frame_idx = 0
    for i in range(0, len(paths), N):
        group = paths[i:i+N]
        if len(group) < N: break
        img = composite(group, weights)
        out = os.path.join(OUTPUT_DIR, f"frame_{frame_idx:05d}.png")
        img.save(out, compress_level=1)
        writer.writerow([frame_idx, ";".join(group), ";".join(map(str, normalize(weights)))])
        frame_idx += 1

# Stitch to video (25 fps)
cmd = ["ffmpeg", "-y", "-framerate", "25", "-i", os.path.join(OUTPUT_DIR, "frame_%05d.png"),
       "-c:v", "libx264", "-pix_fmt", "yuv420p", "-r", "25", "out.mp4"]
subprocess.run(cmd, check=True)
print("Wrote out.mp4")
```

FFmpeg command and image‑sequence behaviour documented in the official manual. [[image2 demuxer](https://ffmpeg.org/ffmpeg-formats.html#image2-1), [ffmpeg docs](https://ffmpeg.org/ffmpeg.html)]

### B.3 Calling *Processing* from Python

From Python you can invoke `processing-java` via `subprocess`, passing arguments or Java properties, enabling you to reuse existing Processing sketches within a notebook pipeline.

```
import subprocess, os

sketch = "/workspace/sketch"  # adjust path
env = os.environ.copy()
env["JAVA_TOOL_OPTIONS"] = "-DINPUT_DIR=/data/in -DOUTPUT_DIR=/data/out -DW=1920 -DH=1080"

cmd = ["xvfb-run", "processing-java",
       f"--sketch={sketch}", "--output=/workspace/build", "--force", "--run"]
subprocess.run(cmd, check=True, env=env)
```

CLI options & arg‑passing are covered in the Processing wiki. [[Command Line](https://github.com/processing/processing/wiki/Command-Line)]

## C. Suggested Pipeline & Storage

1. Stage input images (UHD PNG/JPG) into `/data/in`; generate a run ID.
2. Run the Dockerized sketch (*or* Python notebook) to produce composites/frames.
3. Write `manifest.csv` capturing parameters (strategy, weights, stride, source files).
4. Stitch frames with FFmpeg to target fps (24/25/30/60 as needed).
5. Upload outputs (`.png`, `.mp4`, `manifest.csv`) to cloud storage (GCS/S3) with run‑ID folders.

## D. Gotchas & Performance Tips

- **Headless:** If a sketch uses AWT/OpenGL, wrap with `xvfb-run`; for pure off‑screen `PGraphics` you often won’t need a visible window. [[Headless wiki](https://github.com/processing/processing/wiki/Running-without-a-Display)]
- **Off‑screen correctness:** Always call `beginDraw()`/`endDraw()` on `PGraphics`, then `save()`. [[PGraphics reference](https://processing.org/reference/pgraphics)]
- **Video compatibility:** Use `-pix_fmt yuv420p` for broad playback and set `-framerate` on input. [[FFmpeg docs](https://ffmpeg.org/ffmpeg.html)]
- **Throughput:** Run multiple containers with different input shards; keep PNG compression low (`compress_level=1`) during batching, recompress later if needed.
- **Reproducibility:** Log *all* parameters into `manifest.csv`; version your container.
