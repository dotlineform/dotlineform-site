# Processing → Structure all sketches into one cohesive application

This guide shows how to merge multiple Processing sketches (permutations, composites, batch interpolation, FFmpeg overlay GUI, etc.) into a single modular desktop app that you can export once and keep extending.

## 1) Recommended folder layout

```
processing-app/
├─ app/                      # the main "launcher" sketch (one PApplet)
│  ├─ AppMain.pde            # entry point (setup/draw), state machine/router
│  ├─ AppContext.pde         # shared services (config, paths, caches)
│  ├─ AppMenu.pde            # keyboard/GUI menu to switch tools
│  └─ events/                # simple event bus (optional)
│
├─ core/                     # reusable utilities (no app-specific logic)
│  ├─ io/                    # loading, saving, folder pickers, CSV helpers
│  ├─ gfx/                   # image math, blend modes, auto-levels
│  ├─ ui/                    # lightweight UI controls (buttons, sliders)
│  ├─ math/                  # noise, easing, color utils, geometry
│  └─ video/                 # wrappers (FFmpeg command builder, Movie helpers)
│
├─ tools/                    # each feature/screen = one Tool module
│  ├─ permutations/          # e.g., layered composites from permutations
│  │  ├─ PermutationsTool.pde
│  │  └─ PermutationsEngine.pde
│  ├─ composites/            # simple N-layer composite, UHD-safe pipeline
│  │  └─ CompositesTool.pde
│  ├─ interpolator/          # Batch image interpolation (concept)
│  │  └─ InterpolatorTool.pde
│  └─ ffmpeg_overlay/        # Processing GUI that builds FFmpeg commands
│     └─ FFmpegOverlayTool.pde
│
├─ data/                     # default Processing data folder
│  ├─ config.yaml            # global settings (paths, defaults)
│  ├─ ui_theme.json          # colors, margins, sizes
│  ├─ shaders/               # .glsl if used
│  └─ demo/                  # small sample assets
│
├─ libs/                     # external jars or contributed libs (if any)
│
├─ export/                   # app writes outputs here by default
│  ├─ images/
│  ├─ video/
│  └─ logs/
│
└─ build-notes/              # notes + scripts for Export Application
   └─ gradle-bridge.md       # optional: IntelliJ/Gradle pathway
```

## 2) Core design pattern: a tiny “Tool” interface + router

Each module implements the same interface so the main app can switch tools at runtime.

```
// tools/Tool.pde
interface Tool {{
  String id();                   // stable id: "composites", "permutations", etc.
  String label();                // menu label
  void init(AppContext ctx);     // called once when tool becomes active
  void update();                 // per-frame logic
  void render();                 // draw UI + canvas
  void keyPressed();             // optional input handlers
  void mousePressed();
  void shutdown();               // cleanup when switching tools
}}
```

```
// app/AppMain.pde
AppContext ctx;
Tool[] tools;
int active = 0;

void settings(){{ size(1600, 900, P2D); }}
void setup(){{
  surface.setTitle("Processing Suite — Unified App");
  ctx = new AppContext(this);
  tools = new Tool[] {{
    new CompositesTool(),
    new PermutationsTool(),
    new InterpolatorTool(),
    new FFmpegOverlayTool()
  }};
  tools[active].init(ctx);
}}

void draw(){{
  background(14);
  tools[active].update();
  tools[active].render();
  AppMenu.draw(this, tools, active);
}}

void keyPressed(){{
  if (key == TAB) switchTool((active+1) % tools.length);
  else tools[active].keyPressed();
}}

void mousePressed(){{
  tools[active].mousePressed();
}}

void switchTool(int idx){{
  tools[active].shutdown();
  active = idx;
  tools[active].init(ctx);
}}
```

## 3) AppContext: one place for shared services

```
// app/AppContext.pde
class AppContext {{
  final PApplet p;
  File sourceDir, outputDir;
  Config config;
  UITheme theme;
  Cache cache = new Cache();

  AppContext(PApplet p){{
    this.p = p;
    this.config = Config.load(p.sketchPath("data/config.yaml"));
    this.theme  = UITheme.load(p.sketchPath("data/ui_theme.json"));
    this.outputDir = new File(p.sketchPath("export"));
    if (!this.outputDir.exists()) this.outputDir.mkdirs();
    this.sourceDir = new File(System.getProperty("user.home")); // default
  }}
}}
```

## 4) Minimal UI menu

```
// app/AppMenu.pde
static class AppMenu {{
  static void draw(PApplet p, Tool[] tools, int active){{
    p.pushStyle();
    p.noStroke(); p.fill(18, 200);
    p.rect(0, 0, p.width, 28);
    p.fill(220);
    p.textAlign(LEFT, CENTER);
    float x = 12;
    for (int i = 0; i < tools.length; i++) {{
      String t = (i == active ? "● " : "○ ") + tools[i].label();
      p.text(t, x, 14);
      x += p.textWidth(t) + 18;
    }}
    p.popStyle();
  }}
}}
```

## 7) Performance notes (UHD & video)

- **UHD stills**: Use tiling or a pyramid preview (e.g., 0.25x on-canvas, 1.0x export). Avoid holding many full-res `PImage`s.
- **Batch**: Run a job queue; keep draw() responsive; write `manifest.csv` with parameters + sources.
- **Video**: If you don’t need pixel-by-pixel Processing, prefer FFmpeg via a dedicated tool (faster, lower memory).

## 8) Exporting the unified app

1. Open **app/AppMain** in the Processing IDE
2. Ensure Mode = Java; Renderer = P2D
3. **Sketch → Export Application…** and select target platforms
4. Bundle `data/`; keep paths via `sketchPath()`

## 9) Migration steps

1. Identify shared code → move to `core/`
2. Wrap each old sketch as a `Tool` (init/update/render)
3. Centralise config/paths in `AppContext`
4. Unify UI elements in `core/ui`
