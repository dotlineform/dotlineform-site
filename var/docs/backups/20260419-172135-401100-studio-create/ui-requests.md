# UI Requests

This section stores UI-driven feature requests, working specs, and task breakdowns.

Use it for:

- UI ideas captured from IAB review
- feature specs for new interface behavior
- task docs that turn a UI request into an implementation plan
- follow-on UI request docs that should stay visible without being merged immediately into the stable framework docs

Do not use it for:

- stable design-system rules
- long-term Studio UI standards
- runtime data-model ownership
- script usage or build mechanics

Those still belong in:

- [Design](/docs/?scope=studio&doc=design)
- [Studio UI Rules And Decision Log](/docs/?scope=studio&doc=studio-ui-rules)
- [Docs Viewer](/docs/?scope=studio&doc=docs-viewer)
- [Scripts](/docs/?scope=studio&doc=scripts)

## Working Pattern

For a new UI request, prefer creating:

1. one spec doc that defines the desired behavior
2. one task doc that breaks the work into implementation steps

Recommended naming pattern:

- `<feature>-spec`
- `<feature>-task`
